# 完成汇报格式

> 触发：任务级别完成（AGENTS.md 第 12 条）。每个任务包执行完都要按此格式汇报。
> 不要与会话结束清单混淆——这是单任务汇报，那是会话级文档收尾。

---

## 模板

```text
- [ ] 报告已审阅，可归档

任务包：<任务包文件名>
报告时间：YYYY-MM-DD HH:MM +08（精确到分钟+时区）
耗时：<X 小时 Y 分钟>（墙钟时间，从任务开始到报告完成的实际耗时）
Token 消耗：<输入 N + 输出 M = 总计 T>（如无法获取，标注"未记录"并说明原因）

任务理解：
- <本次任务的目标和边界，用自己的话复述>

完成内容：
- <按文件或模块列出实际修改>

验证结果：
- <列出实际运行过的命令，并贴关键输出>
- <人工验证步骤及结果，如适用>

剩余风险：
- <未覆盖的边界、已知 TODO、潜在退化>
- <无新风险时写："未发现新的剩余风险"，不要省略此节>

后续建议：
- <仅限和本任务直接相关的建议>
- <不要在这里提出新需求、新范围、新架构想法——那些应走任务包或 ADR>
```

---

## 时间精度要求（2026-05-12 起）

报告类文档（任务完成报告、收尾报告、阶段报告）的所有时间字段必须**精确到分钟+时区**：

- 报告头部"报告时间" → `2026-05-12 02:08 +08`
- 任务执行起止时间 → `开始 2026-05-12 00:35 +08 / 完成 2026-05-12 02:08 +08`
- 验证命令的运行时间（如关键性能或长耗任务）

**理由：** 同一天可能有多次任务或多个 Agent 协作，只写日期无法分辨先后顺序和实际工作时长。

---

## 成本统计要求（2026-05-12 起）

任务完成报告和日报必须记录两项成本指标：

| 指标 | 说明 | 示例 |
|------|------|------|
| **耗时** | 墙钟时间（wall-clock），从任务/会话开始到完成的实际分钟/小时数 | `~2 小时 15 分钟`、`45 分钟` |
| **Token 消耗** | 输入 / 输出 / 总计（单位 K 或 M） | `输入 85K + 输出 12K = 总计 97K` |

**Token 获取方式：**
- Claude Code：会话结束时终端输出 `Total tokens: NNN,NNN`
- 如会话仍在进行中无法获取精确值 → 标注 `估算 ~NNK` 或 `未记录（会话中断）`
- 禁止编造精确数字

**理由：** 无法度量就无法优化。记录成本是后续评估"这个任务值不值得用 AI 做"和"模型选型（Opus vs Sonnet vs Haiku）"的基础数据。

## 反模式

| 反模式 | 为什么不行 |
|--------|-----------|
| 用"代码已修改"代替"完成内容" | 不可追溯，看不出改了什么 |
| 用"测试通过"代替"验证结果" | 没有命令、没有输出，无法复核 |
| 跳过"剩余风险" | 违反 AGENTS.md 第 9 条（不隐藏不确定点） |
| 在"后续建议"里夹带新功能 | 应该走任务包流程，不是混在汇报里 |
| 没运行验证命令就声明完成 | 违反 AGENTS.md 第 11 条 |
| 产出物缺审阅 checkbox | 人类无法确认 AI 产出是否已过目 |

## 与其他文档的关系

- **任务包**（[docs/tasks/TEMPLATE.md](../tasks/TEMPLATE.md)）定义了"该做什么"
- **完成汇报**回答"做了什么、验证了什么、还有什么风险"
- **日报**（[journal-spec.md](journal-spec.md)）汇总当天所有任务的成果
- **ADR**（[../decisions/README.md](../decisions/README.md)）记录其中的重大决策
