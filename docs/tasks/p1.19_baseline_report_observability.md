# 任务包：P1.19 baseline 报告可观测性修正

**任务包版本：** v1
**创建时间：** 2026-05-15 11:05 +08

## 任务名称
P1.19 baseline 报告可观测性修正

## 任务类型
fix / feat / test

## 当前阶段
Phase 1 运行观察期

## 来源任务
用户反馈：baseline 报告中部分字段为 N/A，检测时间语义混淆，且缺少从 A 库到报告结果的漏斗数据。

## 目标
修正 baseline 新季写入和导出报告，让 baseline 检测时间、事件写入时间、A 库当前季数和漏斗数据可区分、可解释。

## 非目标
- ❌ 不修改 A 库数据。
- ❌ 不重新设计 `content_updates` schema。
- ❌ 不改变 `baseline-track` 的轮询计划语义。
- ❌ 不修复历史事件中缺失 TMDb 状态等旧字段，只做兼容展示。

## 允许修改范围
- `src/movietrace/pipeline/baseline_tracking.py`
- `src/movietrace/reports/export_writer.py`
- `tests/pipeline/test_baseline_tracking.py`
- `tests/reports/test_export_writer.py`
- `docs/tasks/p1.19_baseline_report_observability.md`
- `STATE.md`

## 禁止修改范围
- 🚫 数据库 schema / migration
- 🚫 `AGENTS.md`、`CLAUDE.md`
- 🚫 外部 API 客户端行为

## 相关上下文
- P1.18 已将 `daily-discover` 与 `baseline-track` 拆分。
- `content_updates.created_at` 是事件写入时间；baseline 检测时间应来自 baseline 运行自身。
- 现有 `source_summary_json.detected_at` 已记录 baseline 检测时间，但报告未使用。

## 输入
- `virtual_series`
- `canonical_items`
- `upstream_programs`
- `content_updates`

## 输出
- baseline `source_summary_json` 增加清晰字段。
- baseline Markdown/JSON 报告增加时间区分、A 库当前季数、漏斗数据。

## 具体要求
1. 新写入的 baseline `new_season` 事件必须写入 `baseline_detected_at` 与 `event_written_at` 可区分字段。
2. 新写入的 baseline `new_season` 事件必须写入 A 库当前季数：`baseline_local_max_season`。
3. 报告表格展示 A 库当前季数、TMDb 当前季数、baseline 检测时间、事件写入时间。
4. 报告顶部展示 baseline 漏斗数据：A 库原始节目、在线节目、疑似季级 TV、B 库 TV season、聚合剧集、可追踪剧集、当前报告结果。
5. 历史事件兼容：旧 `detected_at` 可作为 `baseline_detected_at` fallback；旧事件缺少 A 库当前季数时可由 `season_min - 1` 推断。

## 验收标准
1. ✅ baseline 报告不再把 `created_at` 单独标为“检测时间”。
2. ✅ baseline 报告顶部有漏斗数据。
3. ✅ baseline 报告行内包含 A 库当前季数。
4. ✅ JSON 导出包含新增字段。

## 测试要求
- 更新 baseline 写入测试，验证新增 summary 字段。
- 更新导出测试，验证 Markdown/JSON 包含漏斗和新增字段。

## 验证命令
```bash
PYTHONPATH=src python -m pytest tests/pipeline/test_baseline_tracking.py tests/reports/test_export_writer.py -v
PYTHONPATH=src python -m pytest tests/ -v
```

## 风险点
- 历史 `new_season` 事件缺少新字段，报告只能兼容推断，不能还原所有历史事实。
- A 库疑似 TV season 的统计依赖命名规则 `Sxx`，是当前数据模型下的近似漏斗口径。

## 完成后输出要求
按 [docs/workflow/report-format.md](../workflow/report-format.md) 格式汇报。
