# 任务包：P1.57f export-current-discovery

**任务包版本：** v1

## 任务名称
P1.57f export-recommendations 改读 current discovery items

## 任务类型
feat

## 当前阶段
Phase 1（V1 运行观察期）

## 来源任务
ADR-0016 要求飞书热点发现表同步当前发现项，不再把每日 discovery 事件当运营工作台；导出 JSON 是飞书同步的输入，因此必须先切换导出语义。

## 目标
让 `export-recommendations` 对 discovery 部分读取 current discovery items，并保留 `new_season` 从 `content_updates` 读取的既有语义。

## 非目标
- ❌ 不改飞书同步 upsert key
- ❌ 不停止 `content_updates new_discovery` 写入
- ❌ 不改 `baseline-track`
- ❌ 不新增飞书字段
- ❌ 不改评分公式

## 允许修改范围

- `src/movietrace/reports/inspect_renderer.py`（仅限 inspect 输出适配必需）
- `src/movietrace/cli.py`（仅限 export/inspect 参数或调用适配）
- `tests/test_export*.py`
- `tests/test_inspect*.py`
- `STATE.md`

## 禁止修改范围
- 🚫 `src/movietrace/feishu/**`
- 🚫 `src/movietrace/pipeline/discovery.py`
- 🚫 `src/movietrace/db/migrations/**`

- 🚫 `AGENTS.md`、`CLAUDE.md`、`SCOPE.md`

## 相关上下文
- `reports/latest.json` 当前是 `sync-feishu-table` 的输入。
- 新 JSON 需要兼容 sync 现有字段，同时增加 current 语义字段供 P1.57h 使用。
- 当前 `_load_content_updates`（`export_writer.py:108-183`）通过 `report_kind` 控制 `cu.update_type` 过滤，再 join `canonical_items` / `virtual_series` / `upstream_episodes` / `api_cache`，并以 `content_update_id` 解析 `discovery:type:tmdb_id:date` 反向取季数 / imdb_id。改造时 discovery 部分必须**完全改读 `current_discovery_items` join `canonical_items`**，不再走 `content_updates`，避免双写期 `content_update_id` 两种格式并存导致重复行或解析分叉。`new_season` 部分维持原 `_load_content_updates` 路径。

## 输入
- `current_discovery_items`
- `discovery_observations`
- `content_updates` 中的 `new_season`

## 输出
- `reports/latest.md`
- `reports/latest.json`
- 更新后的导出测试

## 具体要求
1. discovery 行使用稳定 `content_update_id` / `discovery_key`：`discovery:{movie|tv}:{tmdb_id}`，不含日期。
2. JSON 增加或保留字段：首次发现日期、最近发现日期、发现次数、最新分数、最新优先级、source summary。
3. `new_season` 继续从 `content_updates` 导出，不受影响；discovery 必须**完全改读 `current_discovery_items`**，不再 join `content_updates`，从根上消除双写期 `discovery:t:id` vs `discovery:t:id:date` 两种格式并存的去重难题。
4. `--days` 对 discovery 的语义改为"最近 N 天内被观察到的当前发现项"（按 `current_discovery_items.last_discovered_date` 过滤）；对 `new_season` 保持原事件窗口（`content_updates.created_at`）。
5. discovery 行的 `tmdb_id` / 季数 / imdb_id 从 `current_discovery_items.stable_metadata_json` 或 join `canonical_items` 取，不再依赖 `_extract_tmdb_id_from_discovery_id` 这类按日期格式解析的 helper；该 helper 在 P1.57i 后可清理，但本任务暂保留以防过渡期破坏 inspect 报告。
6. MD 中同一 discovery 只出现一行（current 表唯一性自然保证）。

## 验收标准
1. ✅ 同一 discovery 内容跨天多次 observation，导出只出现一行。
2. ✅ `discovery_count` 和最近发现日期来自 current item。
3. ✅ `new_season` 导出保持现状。
4. ✅ `reports/latest.json` 能被现有 sync 命令读取，不因字段缺失崩溃。
