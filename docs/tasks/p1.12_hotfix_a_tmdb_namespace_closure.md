## 来源任务
2026-05-14 Codex 全面代码 review：P1.9-hotfix-E 已引入 `tv:{id}` / `movie:{id}`，但查询和部分写入路径仍未闭环。

## 目标
确保 TMDb movie/tv ID 在所有读写路径中严格按 media_type 隔离，不再出现 TV 查询命中 `movie:{id}`、新写入未加前缀、或将 `tv:{id}` 传给 TMDb API 的情况。

## 非目标
- ❌ 不重新设计 `external_ids` 表结构。
- ❌ 不处理 Trakt / OMDb / upstream 的 external_id 命名空间。
- ❌ 不改评分、merge 或报告逻辑。
- ❌ 不做新集更新追踪。

## 相关上下文
- Migration 011 已把历史 TMDb external_id 迁移为 `tv:{id}` / `movie:{id}` / `unknown:{id}`。
- 当前 `_lookup_canonical_id(conn, tmdb_id, media_type="tv")` 仍会 fallback 查询 `movie:{id}` 和裸 ID。
- `match_upstream_program()` 仍写入裸 TMDb ID。
- `virtual_series` 回填路径读取到 `tv:{id}` 时，会把带前缀字符串传给 `get_tv_details()`。

## 输入
- 当前 DB schema version 12
- 现有 `external_ids` 数据
- P1.9-hotfix-E 任务包和 migration 011

## 输出
- 严格按 media_type 查询的 `_lookup_canonical_id()`
- 全部 TMDb external_id 新写入统一带前缀
- virtual_series 回填能兼容带前缀和历史裸 ID
- 如生产 DB 可能已有 post-011 裸 ID，提供 data-only migration 013 清理

## 具体要求
1. `_lookup_canonical_id(conn, tmdb_id, media_type)` 只允许查询当前 media_type 对应的前缀：
   - `media_type in ("tv", "show")` → 只查 `tv:{id}`
   - `media_type == "movie"` → 只查 `movie:{id}`
   - 历史裸 ID 兼容仅允许在明确 migration/测试证明不会跨类型错链时使用；默认不跨类型 fallback。
2. `_ensure_canonical_item()` 保持写入 `tv:{id}` / `movie:{id}`。
3. `match_upstream_program()` 写入 `source='tmdb'` 时必须写前缀 external_id，并正确设置 `external_granularity`。
4. `find_or_create_virtual_series_for_canonical_item()` 和回填脚本读取 `external_ids.external_id` 后，必须剥离 `tv:` 前缀再调用 TMDb TV detail。
5. 如新增 migration 013：
   - 只做数据清理，不改表结构。
   - 将 post-011 新产生的裸 TMDb ID 重新规范化。
   - `schema.py::SCHEMA_VERSION` 同步更新。

## 验收标准
1. ✅ 当 DB 只有 `movie:100` 时，`_lookup_canonical_id(conn, 100, "tv")` 返回 `None`。
2. ✅ 当 DB 只有 `tv:100` 时，`_lookup_canonical_id(conn, 100, "movie")` 返回 `None`。
3. ✅ `match_upstream_program()` 新写入的 TMDb external_id 带正确前缀。
4. ✅ virtual_series 回填不会把 `tv:{id}` 原样传给 TMDb API。
5. ✅ 全量测试通过。
