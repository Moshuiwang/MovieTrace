## 背景

- V1 时期 `canonical_items` 的输入是飞书 `baseline_items`（~289 条 TV + ~若干电影），匹配覆盖不完整
- 现在 A 库 `upstream_programs` 有 **735 条节目**，是完整的内容目录
- 需要全量重新匹配，让 `canonical_items` 成为 A 库的完整 TMDb 映射
- 以后 A 库数据更新（通过 API 增量同步）时，增量匹配由此模块复用

### A 库数据结构提醒

| 字段 | 含义 | 匹配中的角色 |
|------|------|------------|
| `id` | 节目 ID | → `external_id` (source='upstream') |
| `name` | 节目名称 | TMDb 搜索输入。如 "Better Call Saul S01"、"Avatar The Way of Water" |
| `online_flag` | 上架状态 | 过滤：只匹配已上架(1)的节目 |

### 关键设计约束

- **`program_status` 不可靠** — 734/735 都是 MOVIE，不能用来判断电影 vs 剧集。改为从 `name` 含不含 `S\d\d` 推断
- **`imdb_id` 全空** — 不能走 ID 直连，只能走名称搜索
- **季号在 name 里** — 如 "Better Call Saul S01" → season_number=1，需在匹配前剥离季号做 series 级搜索

## 目标

1. 为 A 库 735 条节目全部创建 `canonical_items` + TMDb `external_ids`
2. 匹配策略：**从 name 提取 series 名 + season_number → TMDb 搜索 series → 找对应季**

4. 低置信度记录不阻塞流程，记入 `baseline_quality_issues`

## 输出

### 1. 匹配入口函数（追加到 `src/movietrace/pipeline/entity_matching.py`）

```python
def match_upstream_program(
    conn: sqlite3.Connection,
    upstream_program_id: int,
    tmdb_client: TmdbSearchClient,
) -> dict | None:
    """对单条 upstream_program 执行 TMDb 匹配，返回匹配结果。

    流程：
    1. 从 upstream_programs 读取 name、online_flag 等
    2. 正则判断 name 是否含 S\d\d：
       - 含 → 视为 TV season。剥离季号得到 series_name，搜 TMDb /search/tv
       - 不含 → 视为 movie。直接搜 TMDb /search/movie
    3. 候选评分（复用 V1 评分逻辑）
    4. 选最佳匹配 → 创建 canonical_item + external_id
    5. 低置信度 → 记录 baseline_quality_issues，仍创建 canonical_item（标记 match_confidence_low）

    返回 {'canonical_item_id': int, 'confidence': str, 'tmdb_id': str, ...}
    """
```

### 2. 全量匹配脚本（`scripts/p1.5_e_match_all.py`）

```
流程：
1. 连接 B 库
2. 读取 upstream_programs WHERE online_flag='1' AND id NOT IN (
     select external_id from external_ids where source='upstream'
   )
   （跳过已匹配的，支持中断重启）
3. 对每条：
   a. 调 match_upstream_program()

   - 成功匹配数 / 低置信度数 / 失败数

CLI：python scripts/p1.5_e_match_all.py [--dry-run] [--limit N] [--db path]
```

## 适配现有 entity_matching 模块的注意事项

V1 的 `entity_matching.py` 从 `baseline_items` 读取输入。本任务需要新增一条从 `upstream_programs` 读取的路径：

- **保持现有函数签名不动**（V1 代码不受影响）

- **季号剥离逻辑**：用正则 `S\d{2}` 从 name 提取 season_number，剩余部分作为 series 搜索词

## 风险点

1. **TMDb 搜索配额** — 597 次搜索 + 597 次详情（如果候选合并需要）≈ 最多 1200 次调用。1 次/秒约 20 分钟。对策：限流 1 次/秒，打印进度和剩余时间估算。
2. **季号剥离不准** — 部分 name 可能不含 `S\d\d` 却是 TV 剧集（如日剧罗马音标题），正则提取失败会错判为 movie。对策：记入 baseline_quality_issues，V1.6 观察后人工修复。
3. **与现有 canonical_items 冲突** — B 库已有部分 canonical_items（从 V1 飞书匹配来的 ~289 条）。对策：脚本执行前先检查是否要清空重建（建议全量重建，因为数据源已从飞书切换到 A 库）。
4. **同名不同片** — 如 "Friends"（NBC 1994）vs "Friends"（韩剧 2001）。对策：依赖 TMDb 搜索的 year/popularity 排序，首选最热门结果；模糊案例归入低置信度。
