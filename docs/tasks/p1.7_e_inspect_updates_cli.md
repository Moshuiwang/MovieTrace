## 具体要求

### CLI 接口

```bash
# 列出最近 N 天的所有 updates(默认 7)
PYTHONPATH=src python -m movietrace.cli inspect-updates [--days N]

# 按优先级过滤
PYTHONPATH=src python -m movietrace.cli inspect-updates --priority P0
PYTHONPATH=src python -m movietrace.cli inspect-updates --priority P0,P1

# 按 update_type 过滤
PYTHONPATH=src python -m movietrace.cli inspect-updates --type new_discovery
PYTHONPATH=src python -m movietrace.cli inspect-updates --type new_season

# 单条详情
PYTHONPATH=src python -m movietrace.cli inspect-updates --id <content_update_id>

# 输出格式
PYTHONPATH=src python -m movietrace.cli inspect-updates --format table  # 默认,终端表格
PYTHONPATH=src python -m movietrace.cli inspect-updates --format json   # JSON 输出
PYTHONPATH=src python -m movietrace.cli inspect-updates --format md     # 写入 reports/inspect_*.md
```

- FlixPatrol: Netflix US #1, 32天在榜
- TMDb: popularity=569.2, vote_average=8.45 (12,247 votes)
- Trakt: watchers=5520, rating=8.38 (36,983 votes)

### 实现要点

- `inspect_renderer.py` 不依赖外部库(tabulate 等),用 f-string 手工对齐
- 表格列宽自适应:title 最多 30 字符,超出加省略号
- JSON 输出复用 `export_writer.format_json` 接口
- `source_summary_json` 解析容错(可能为空 / 损坏)

## 风险点

- **数据量不足**:如果当日 FP/TMDb 数据稀少,可能只有 10-20 条 P2+,表格看起来不丰富。验收报告应说明数据规模
- **`source_summary_json` 格式不一致**:不同条目可能有不同的字段(比如有的没 Trakt 数据),渲染逻辑要容错
- **inspect 在 P1.7-D 之前无意义**:这是 Phase 1.7 收尾任务,不能独立做
