   ```python
       """'1190634' → 'tt1190634', 'tt1190634' → 'tt1190634'."""
       cleaned = str(raw).strip()
       if cleaned.lower().startswith("tt"):
           digits = cleaned[2:]
       else:
           digits = cleaned
       digits = digits.zfill(7)
       return f"tt{digits}"
   ```

## 风险点

- **`zfill(7)` 对短数字补零**：5/6 位 imdb_id 会被补 0 到 7 位。这是 IMDb 官方约定（`tt0000001`），但**如果数据库里存的 5/6 位 ID 实际不是去掉前导 0 的合法 ID 而是脏数据**，补 0 后访问也是 404。
  - 数据现状：5/6 位共 14 个 ID，本任务包不抽样验证（不在范围内）；如运营反馈仍有 404，再开 issue 处理。

- **8 位 ID 不被截断**：已通过 Python `zfill` 行为确认（`"31589662".zfill(7) == "31589662"`）。但 IMDb 未来若发布 9 位 ID，需重新评估。**本任务不预设 9 位兼容**。

- **历史飞书表脏数据**：74 行旧值不会主动被覆盖，但下次有同 `content_update_id` 的 daily-discover 会触发飞书 upsert 自然刷新。**本任务包不做强制刷新**。
