## 风险点

- **风险 1：词汇全局替换可能误伤**。"推荐"在"实体匹配推荐合并"等算法术语中也存在，全局替换需要逐处审视，**不能一刀切**。已在"具体要求 § F"列出例外规则。
- **风险 2：R9 / R10 章节保留的方式**。决定保留作为历史，但顶部加注"自 V1.5 起移出系统职责范围"。如果用户希望直接删除，需在本任务包 review 阶段澄清。
- **风险 3：R13 新需求的字段细节**。R13 涉及 virtual_series 表，但本任务包只在文档层描述需求，不实现 schema。具体字段在 P1.5-C 任务包细化。本任务包中 R13 应按"输入/输出/验收标准"风格写就，不写 SQL DDL。
- **风险 4：product_roadmap.md § 2 V1 路线** 大段术语替换的工作量可能较大，需要逐段重写。
- **风险 5：历史飞书表的实际字段集**。本任务只改文档，但 P1.5-B 实现 schema migration 时可能发现现有飞书表中字段名与文档不完全一致（如飞书字段中英文混用）。本任务包不处理这个，但需要记录"实际飞书表字段集需在 P1.5-B 阶段重新核对"。
