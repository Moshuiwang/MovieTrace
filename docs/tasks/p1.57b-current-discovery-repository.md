新增最小 current discovery 读写 helper，提供稳定键构造、当前项 upsert、observation upsert 和查询能力。

## 非目标

- ❌ 不改导出或飞书同步

- ❌ 不处理 enrichment skip 策略

## 相关上下文
- P1.57a 已创建 `current_discovery_items` 和 `discovery_observations`。
- 项目无 ORM，helper 应保持函数式、显式传入 sqlite connection。

## 输入
- P1.57a schema
- ADR-0016 稳定键和 observation 语义

## 输出
- `build_discovery_key(content_type, tmdb_id)`
- `upsert_current_discovery_item(...)`
- `upsert_discovery_observation(...)`
- `get_current_discovery_item(...)` 或等价查询 helper
- 单元测试

## 具体要求
1. 稳定键必须是 `discovery:{movie|tv}:{tmdb_id}`，输入非法时显式失败。
2. 当前项首次写入设置 `first_discovered_date`，后续更新只更新最近日期、发现次数、最新分数/优先级/来源摘要。
3. observation 按 `(discovery_key, observed_date)` 幂等 upsert。
4. helper 不自行打开数据库连接，不提交外层事务；由调用方控制事务边界。
5. JSON 字段由 helper 统一 `json.dumps(..., ensure_ascii=False)`，避免下游重复编码。

## 验收标准
1. ✅ 首次 upsert 创建 current item，发现次数为 1。
2. ✅ 次日 observation upsert 后 current item 的最近发现日期和发现次数正确更新。
3. ✅ 同日重复 observation 不增加发现次数。
4. ✅ 非法 content type / 缺失 tmdb_id 会失败，不静默写脏数据。

## 测试要求
- 使用临时 SQLite DB 初始化 v19 schema。
- 覆盖首次写、同日幂等、跨日更新、非法输入。
