## 任务名称
P1.57d 固化 discovery observation 写入资格判断

## 任务类型
refactor

## 当前阶段
Phase 1（V1 运行观察期）

## 来源任务
ADR-0016 规定纯 source-date fallback 不计入有效 observation；P1.56 也指出 write gate 需要集中 invariant，防止历史 source snapshot 被误当作今日新发现。

## 目标
新增集中判断，明确候选是否允许写 current discovery / observation。

## 非目标
- ❌ 不写入 current discovery 表
- ❌ 不改 enrichment
- ❌ 不改导出或飞书同步
- ❌ 不做完整 P1.56 source decision 重构
- ❌ 不改变评分公式

## 相关上下文
- P1.42 已修复 pure fallback 写 `content_updates` 的污染，但**保留了 Soap 豁免**：当前 `discovery.py:440` 行 `passed = [c for c in passed if c.get("has_fresh_signal") or c.get("is_soap")]`，即 `is_soap=True` 即便 pure source-date fallback 也能通过 write gate。
- P1.57d 要把该判断升级为 observation/current item 资格，而不只是旧 `content_updates` 写入保护。
- ADR-0016 明确"默认不允许 pure fallback Soap"——这是相对于 P1.42 现状的**有意行为变更**：从今往后，Soap 内容必须有 fresh 信号才会被写 observation。某些日报中只靠 fallback 命中的 Soap 会消失，属于预期。

## 输入
- 当前 `run_discovery` stats / source_status / candidate source flags
- P1.56 任务包中的 write gate 要求

## 输出
- `has_fresh_discovery_signal(candidate, ...)` 或等价集中 helper
- `should_write_discovery_observation(candidate, ...)` 或等价集中 helper
- 矩阵测试

## 具体要求
1. pure source-date fallback 候选返回 false。
2. fresh + source-date fallback 混合候选返回 true，fallback 只作为上下文。
3. Soap 豁免**仅作用于阈值（hot_score 门槛）**，不再作用于 fresh signal 闸；pure source-date fallback 的 Soap 候选默认不允许写 observation。
4. helper 名称不得使用裸 `fallback` 表达多义状态，至少区分 source-date fallback。
5. 现有 `content_updates` write gate 行为不得回退（fresh + fallback 混合仍允许写）。
6. stats 中新增 `soap_pure_fallback_suppressed` 计数，与 `suppressed_fallback_only` 区分；便于一周 shadow cron 观测 Soap 变化幅度。

## 验收标准
1. ✅ pure source-date fallback 不允许写 observation。
2. ✅ fresh + fallback 允许写 observation。
3. ✅ Soap pure fallback 不再隐式越过 write gate；stats 中 `soap_pure_fallback_suppressed` 准确计数。
4. ✅ 现有 P1.42 pure fallback suppress 回归测试仍通过。
5. ✅ Soap fresh-signal 候选（即 Soap 有真实采集来源）仍走通完整路径。

## 测试要求
- 覆盖 fresh / pure source-date fallback / mixed / Soap edge case。
- 覆盖 source effective date 与 target date 不一致的场景。
