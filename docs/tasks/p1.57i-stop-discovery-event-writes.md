# 任务包：P1.57i stop-discovery-event-writes

**任务包版本：** v1
**创建时间：** 2026-05-24 00:53 +08

## 任务名称
P1.57i 停止 daily-discover 写每日 new_discovery 事件行

## 任务类型
refactor

## 当前阶段
Phase 1（V1 运行观察期）

## 来源任务
P1.57e-h 已完成 current / observation 双写、导出和飞书稳定键同步。现在可以按 ADR-0016 停止 discovery 每日事件行输出。

## 目标
让 `daily-discover` 不再向 `content_updates` 写入 `update_type='new_discovery'` 的每日事件行；`content_updates` 保留给 `new_season` 和历史兼容。

## 非目标
- ❌ 不删除历史 `content_updates new_discovery`
- ❌ 不删除 `content_updates` 表
- ❌ 不改 `baseline-track` / `new_season`
- ❌ 不改飞书同步字段
- ❌ 不做重复命中 enrichment skip

## 允许修改范围
- `src/movietrace/pipeline/discovery.py`
- `tests/test_discovery*.py`
- `tests/test_export*.py`（仅限兼容断言）
- `STATE.md`

## 禁止修改范围
- 🚫 `src/movietrace/db/migrations/**`
- 🚫 `src/movietrace/feishu/**`
- 🚫 `scripts/**`
- 🚫 dependency / requirements / lock 文件
- 🚫 `AGENTS.md`、`CLAUDE.md`、`SCOPE.md`

## 相关上下文
- ADR-0016 只修正 discovery 类事件输出语义；`new_season` 不在范围内。
- P1.57f 已让导出从 current discovery items 获取 discovery 数据。

## 输入
- P1.57e 双写路径
- P1.57f export current path
- P1.57h sync current path

## 输出
- daily-discover 不再写新 `new_discovery` content_updates
- stats / CLI 文案更新
- 回归测试

## 具体要求
1. 删除或旁路 `new_discovery` 每日 `content_update_id` 生成路径，禁止继续用日期制造 discovery 业务行。
2. CLI 输出不得再显示“写入 content_updates N 条”来描述 discovery current 写入；应区分 current/observation。
3. `content_updates` 中历史 discovery 行保持可查询，不做清理。
4. `baseline-track` 写 `new_season` 不受影响。

## 验收标准
1. ✅ commit 模式运行后 current / observation 增长，`content_updates new_discovery` 不增长。
2. ✅ `content_updates new_season` 测试保持通过。
3. ✅ export/sync 所需 discovery 数据来自 current 表。
4. ✅ pure fallback 仍不会写 observation。

## 测试要求
- 覆盖 daily-discover commit 后 `new_discovery` 行数不增加。
- 覆盖 current / observation 仍更新。
- 覆盖 `new_season` 相关测试不变。

```bash
PYTHONPATH=src python -m pytest tests/ -v --tb=short -q

```

- 这是语义切换点；必须确认 P1.57f-h 已先完成，否则导出/同步会缺 discovery 数据。
