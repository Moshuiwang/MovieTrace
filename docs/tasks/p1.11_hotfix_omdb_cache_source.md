## 目标
修复 OMDb enrichment cache 读写 source 错误，确保 OMDb cache hit 不发真实外部 API 请求。

## 非目标

## 相关上下文
OMDb enrichment 使用 `api_cache` 缓存外部响应。当前测试预置 `source='omdb'`，但实现中的 cache helper 使用了 `source='tmdb'`，导致 OMDb cache miss 后发起真实请求。

## 输入
- 失败测试输出：`1 failed, 437 passed`
- 失败测试：`tests/test_omdb_enrichment.py::OmdbEnrichmentTest::test_enrich_with_omdb_cache_hit`

## 输出
- 修正后的 OMDb cache source 读写逻辑。
- 回归测试明确证明 cache hit 不调用 `OmdbDetailClient.get_by_imdb_id()`。

## 具体要求
- OMDb cache key 仍保持 `omdb:<imdb_id>`。
- OMDb cache 读写必须使用 `api_cache.source = 'omdb'`。
- cache hit 路径不得发起真实 API 调用。
- 保持 TMDb detail cache 行为不变。

## 验收标准
1. ✅ OMDb cache hit 测试通过，且测试会在 API client 被调用时失败。
2. ✅ `tests/test_omdb_enrichment.py` 全部通过。
3. ✅ 全量测试通过。

## 测试要求
新增或更新 `test_enrich_with_omdb_cache_hit`，mock `OmdbDetailClient.get_by_imdb_id` 为一旦调用即失败，防止真实 API 请求回归。
