## 具体要求
1. 新增系统字段建议包括：`首次发现日期`、`最近发现日期`、`发现次数`。
2. 这些字段由程序写入，运营不应手填。
3. `同步批次` 保留系统用途；如运营需要供应商批次，由人工另建字段，程序不写。
4. 字段创建必须幂等，权限不足时按现有规则显式失败。
5. 字段重建只能依赖 `secrets.json` 中的 app/table 配置和飞书 API 返回的字段列表；不得硬编码开发环境字段 ID 或假设字段已存在。

## 验收标准
1. ✅ `ensure_table_fields` 包含新增字段定义。
2. ✅ 单测覆盖字段存在时不重复创建、字段缺失时创建。
3. ✅ `同步批次` 未被删除、重命名或改变类型。

## 风险点
- 飞书字段创建需要开发表授权；权限不足必须报错，不能静默跳过。
- 未来生产飞书表可能与开发表字段状态不同；字段 ensure 必须按名称/类型自动收敛，不能靠人工记忆。
