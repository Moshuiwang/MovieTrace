## 目标

每日运行一次,完成下列采集:

- `/trending/all/day` 前 3 页(60 条)
- `/tv/popular` 前 3 页(60 条)

## 具体要求

### TmdbTrendingClient 接口

```python
class TmdbTrendingClient:
    def __init__(self, bearer_token: str, base_url: str = "..."):
        ...

    def fetch_trending_all_day(self, page: int = 1) -> list[dict]:
        """GET /trending/all/day?page=N&language=en-US"""

    def fetch_tv_popular(self, page: int = 1) -> list[dict]:
        """GET /tv/popular?page=N&language=en-US"""

    def fetch_movie_popular(self, page: int = 1) -> list[dict]:
        """GET /movie/popular?page=N&language=en-US"""
```

每个方法返回的 dict 列表必须包含字段:
`id`, `media_type`(trending返回有,popular无→需补), `title`/`name`, `original_title`/`original_name`, `release_date`/`first_air_date`, `original_language`, `popularity`, `vote_average`, `vote_count`

### 采集编排逻辑(`pipeline/tmdb_trending.py`)

```python
def fetch_and_store_tmdb_trending(
    db_path: str,
    bearer_token: str,
    snapshot_date: str | None = None,
    pages_per_endpoint: int = 3,
) -> dict:
    """
    1. 调三个端点 × pages_per_endpoint 页
    2. 标准化为 tmdb_trending 行
    3. INSERT OR IGNORE(UNIQUE 约束自动去重)
    4. 返回 {fetched, inserted, errors}
    """
```

### 字段标准化规则

- `media_type`:
  - trending/all/day 返回里已有
  - tv/popular → 固定 `tv`
  - movie/popular → 固定 `movie`
- `title`:
  - movie 用 `title`
  - tv 用 `name`
- `release_date`:
  - movie 用 `release_date`
  - tv 用 `first_air_date`
- `original_title`:
  - movie 用 `original_title`
  - tv 用 `original_name`

### 错误处理

- 单页失败:记录日志,继续下一页(不中断整体)
- 全部失败:返回 errors > 0,但不抛异常(便于 daily-discover 容忍部分失败)
- 5xx / 429:重试 1 次,间隔 2 秒

### 礼貌采集

- 端点之间 sleep 1 秒
- 同端点页之间 sleep 0.5 秒

## 风险点

- **页码超过 500 时 API 行为**:TMDb 文档限定 trending/all/day 最大 500 页,本任务只取 3 页无风险
- **TMDb popular 端点重叠**:`tv/popular` 和 `trending/all/day` 里 TV 部分会重叠,UNIQUE 约束按 `(tmdb_id, media_type, source_endpoint, snapshot_date)` 设计,允许同一条目从不同端点入库(查询时按 endpoint 区分,合并时去重)
- **TMDb 配额**:免费 Bearer Token 每秒约 40 次,3 端点 × 3 页 = 9 次/天,完全无压力
