# 任务包：P1.51 fix-double-api-logging

**任务包版本：** v1
**创建时间：** 2026-05-22 22:40 +08

## 任务名称
P1.51 修复 api_usage_log 双重写入——传输层与业务层各写一次导致统计虚高 2×

## 任务类型
fix

## 当前阶段
Phase 1（V1 运行观察期）

## 来源任务
live run 分析发现：`/tv/284838` 在 `api_usage_log` 出现两条记录（status='ok' + status='success'，duration 相差 4ms）。根因是 P1.46 在传输层（`_http_policy.py:_log_transport`）写 `api_usage_log`，而业务层（TMDb client / OMDb client）本身已有独立写入，两层都写。结果：实际 112 次 API 请求产生 224 条 log，OMDb 配额统计虚高 2×。

## 根因详述

```
TMDb client.get_by_tmdb_id()
  └─ sources/http.py:get_json()
       └─ _http_policy.py:request_with_policy()
            └─ _log_transport() → 写 api_usage_log (status='ok')   ← 传输层
  └─ TMDb client 自身 log_api_usage() → 写 api_usage_log (status='success') ← 业务层
```

OMDb client 路径类似。唯一只走传输层不走业务层的是 FlixPatrol（`get_json` 直接用，无额外业务日志）。

## 目标

让每次 HTTP 请求**只写一条** `api_usage_log`：
- 有业务层日志的调用方（TMDb / OMDb）：传输层跳过 DB 写入
- 无业务层日志的调用方（FlixPatrol / 其他直接用 `get_json` 的）：传输层保留写入

## 非目标
- ❌ 不改 `api_usage_log` 表结构
- ❌ 不合并传输层和业务层的日志字段（两者记录不同信息）
- ❌ 不修改 Python `logger` 输出（只管 DB 写入）

## 允许修改范围
- `src/movietrace/sources/_http_policy.py`（`HttpPolicy` 加 `log_to_db: bool` 字段）
- `src/movietrace/sources/http.py`（`get_json` 向 policy 传 `log_to_db`）
- `src/movietrace/sources/tmdb.py`（构造 policy 时设 `log_to_db=False`）
- `src/movietrace/feishu/_http.py`（构造 policy 时视情况设 `log_to_db=False`，飞书已有业务层日志）
- `tests/test_http_policy.py` / `tests/test_http.py`（适配）

## 禁止修改范围
- 🚫 `src/movietrace/sources/omdb.py`（OMDb client 不走 `get_json`，不涉及）
- 🚫 `src/movietrace/db/`
- 🚫 任何业务层日志逻辑

## 具体要求

### 1. `HttpPolicy` 加字段

```python
@dataclass
class HttpPolicy:
    timeout: float = 15.0
    max_retries: int = 3
    backoff_base: float = 1.0
    respect_retry_after: bool = True
    retry_on_status: tuple[int, ...] = (502, 503, 504)
    log_to_db: bool = True          # 新增：False = 跳过 _log_transport 的 DB 写入
```

### 2. `_log_transport` 加守卫

```python
def _log_transport(..., policy: HttpPolicy) -> None:
    if not policy.log_to_db:
        return
    # 原有写 api_usage_log 逻辑不变
```

### 3. 调用方设 `log_to_db=False`

执行前必须读代码确认哪些调用方有自己的业务层日志：
- `sources/tmdb.py`：有 → `log_to_db=False`
- `feishu/_http.py`：有 → `log_to_db=False`
- `sources/http.py` 直接被 FlixPatrol 等调用：无业务层日志 → 保持 `log_to_db=True`（默认）
