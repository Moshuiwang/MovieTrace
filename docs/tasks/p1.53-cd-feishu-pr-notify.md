# 任务包：P1.53 cd-feishu-pr-notify

**任务包版本：** v1
**创建时间：** 2026-05-23 07:20 +08

## 任务名称
P1.53 CD 飞书通知补充 PR 信息

## 任务类型
fix

## 当前阶段
Phase 1（V1 运行观察期）

## 来源任务
用户要求：远端 CD 通过时机器人飞书推送更多本 PR 信息；未通过时也给出相应信息。

## 目标
main 分支 CD 通知无论成功或失败，都带上关联 PR 的编号、标题、作者、链接、merge commit、测试/部署结果和 Actions run 链接。

## 非目标
- ❌ 不修改测试、部署、迁移、secret 同步逻辑
- ❌ 不改变 PR test 阶段是否通知
- ❌ 不引入新依赖

## 允许修改范围
- `.github/workflows/ci.yml`
- `.github/workflows/auto-merge.yml`
- `STATE.md`

## 禁止修改范围
- 🚫 `src/**`
- 🚫 `tests/**`
- 🚫 数据库 schema / migration

## 相关上下文
- `auto-merge.yml` 当前在 PR 合并后用 `gh workflow run CI --ref main` 触发 main CD。
- `ci.yml` 的 `notify` job 当前仅发送 commit、test/deploy 结果和 run 链接。
- PR 事件中的 deploy/notify 正常跳过；main `workflow_dispatch` 才执行 deploy/notify。

## 输入
- GitHub Actions context
- PR number（由 auto-merge dispatch 传入）
- GitHub API

## 输出
- CD 飞书通知文本包含 PR 上下文，成功和失败都可读。

## 具体要求
1. `CI` workflow 增加可选 `workflow_dispatch` input：`pr_number`。
2. `auto-merge.yml` dispatch main CI 时传入 `pr_number`。
3. `ci.yml notify` 使用 `pr_number` 调 GitHub API 获取 PR 详情；无 PR 编号时降级为 commit 信息。
4. 成功文案明确 `部署成功`；失败文案明确 test/deploy 结果。
5. 不泄露 secret。

## 验收标准
1. ✅ `gh workflow run CI --ref main -f pr_number=<PR>` 可触发 main CD。
2. ✅ notify 脚本语法可通过本地 `python -m py_compile` 等价验证。
3. ✅ `gh workflow run` 输入配置可被 GitHub Actions 解析。

## 测试要求
- 本地不跑全量测试；只做 workflow YAML/脚本静态验证。
- 如创建 PR，依赖远端 CI/CD 验证。

## 验证命令
```bash
python - <<'PY'
import pathlib, yaml
for path in [".github/workflows/ci.yml", ".github/workflows/auto-merge.yml"]:
    yaml.safe_load(pathlib.Path(path).read_text())
print("workflow yaml ok")
PY

python -m py_compile /tmp/extracted_notify.py
```

## 风险点
- GitHub Actions 对 `on` 作为 YAML 1.1 布尔值的解析与 PyYAML 不完全一致，本地 YAML 解析只能作为语法烟测。
- 手动触发 CD 时如果不填 `pr_number`，通知会按 commit-only 降级。

## 完成后输出要求
按 [docs/workflow/report-format.md](../workflow/report-format.md) 格式汇报。
