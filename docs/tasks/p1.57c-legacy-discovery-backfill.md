# 任务包：P1.57c legacy-discovery-backfill

**任务包版本：** v1
**创建时间：** 2026-05-24 00:53 +08

## 任务名称
P1.57c 从历史 content_updates 回填 current discovery / observations

## 任务类型
feat

## 当前阶段
Phase 1（V1 运行观察期）

## 来源任务
ADR-0016 要求后续任务明确兼容旧 `content_updates` 数据；P1.57a-b 只提供新表和 helper，尚未把历史 discovery 事件转成当前项和 observation。

## 目标
提供一次性回填脚本，将历史 `content_updates.update_type='new_discovery'` 转换为 current discovery items 和 discovery observations。

## 非目标
- ❌ 不改 `daily-discover`
- ❌ 不改 `content_updates` 历史数据
- ❌ 不同步飞书
- ❌ 不处理 `new_season`
- ❌ 不重新计算历史分数

## 允许修改范围
- `scripts/p1_57_backfill_current_discovery.py`
- `tests/test_p1_57_backfill_current_discovery.py`
- `STATE.md`

## 禁止修改范围
- 🚫 `src/movietrace/db/migrations/**`
- 🚫 `src/movietrace/feishu/**`
- 🚫 `src/movietrace/reports/**`
- 🚫 `src/movietrace/pipeline/discovery.py`
- 🚫 dependency / requirements / lock 文件
- 🚫 `AGENTS.md`、`CLAUDE.md`、`SCOPE.md`

## 相关上下文
- 旧 discovery ID 格式：`discovery:{movie|tv}:{tmdb_id}:{date}`。
- 新稳定键格式：`discovery:{movie|tv}:{tmdb_id}`。
- 历史 `hot_score` 只作为当时规则快照，不新增评分版本。
- P1.42（2026-05-22）后 pure source-date fallback 候选不再写入 `content_updates`；P1.42 之前可能存在 pure-fallback 历史行，本任务**全量回填、不追溯过滤**：这些行视为历史上下文，背离 ADR-0016 "pure fallback 不计入有效 observation" 是已知偏差。脚本必须在汇报中输出 P1.42 切线之前的 observation 计数与样例，使未来分析可剔除。

## 输入
- `content_updates` 历史 `new_discovery` 行
- `source_summary_json`
- P1.57b helper

## 输出
- 可重复运行的回填脚本
- 回填统计：读取行数、current 新建数、current 更新数、observation 写入数、跳过数、错误数
- 单元测试

## 具体要求
1. 脚本默认 dry-run，显式 `--commit` 才写库。
2. 回填前创建 DB 备份，遵循现有备份命名。
3. 无法解析旧 `content_update_id` 的行跳过并计数，不中断整个任务。
4. 同一内容多天历史事件应变成一个 current item + 多条 observation。
5. `first_discovered_date` 取最早 discovery 日期，`last_discovered_date` 取最新 discovery 日期。
6. `discovery_count` 按有效 observation 天数计算，不按重复行数盲加。
7. 回填必须可作为升级流程自动执行：只依赖旧 DB 中的 `content_updates` / `source_summary_json`，不能依赖当前开发环境已知标题清单、手工映射表或飞书表内容。
8. 回填不删除、不改写历史 `content_updates`，失败时应回滚本次 current/observation 写入。
9. 回填统计中必须分别列出 `observations_before_p142_cutoff`（2026-05-22 之前的 observation 行数）与 `observations_after_p142_cutoff`，使运营/分析端可识别 pure-fallback 残留。

## 验收标准
1. ✅ dry-run 不修改数据库但输出准确统计。
2. ✅ commit 后历史 discovery 行可回填为 current + observations。
3. ✅ 重复运行 commit 幂等，数量不翻倍。
4. ✅ `new_season` 行不参与回填。

## 测试要求
- 构造含同一内容多天、多内容、非法旧 ID、`new_season` 的临时 DB。
- 覆盖 dry-run、commit、重复 commit。
