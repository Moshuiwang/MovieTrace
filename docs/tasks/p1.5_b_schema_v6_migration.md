# 任务包：P1.5-B Schema v6 Migration

**版本：** v2（重写，适配 A 库接入 + 飞书移除）
**创建日期：** 2026-05-12 +08
**依赖：** P1.5-A（已完成）、A 库 CSV 导入（已完成，migration 005）

## 任务名称

B 库 schema 升级 v5→v6：新增 `virtual_series` 表、`canonical_items.virtual_series_id`、`content_updates.match_confidence_low`。

## 任务类型

refactor（schema + DDL，不改业务逻辑）

## 当前阶段

Phase 1.5（V1 定位翻转，ADR-0007）

## 背景

- **Migration 005** 已用于 `upstream_programs` / `upstream_episodes`（A 库备份表），已执行完毕
- 本任务包使用 **migration 006**
- **飞书已从系统链路中移除**：飞书不再是内容目录来源（被 A 库取代），也不再是输出渠道（被日报+导出文件取代）
- 现有 `baseline_items` 表（V1 时期从飞书导入）**保留不动**，不在本任务中删除

## 目标

在 B 库中建立 `virtual_series` 聚合层，为 P1.5-C（回填）和 P1.5-D（追踪）打好结构基础。

```
upstream_programs (A 库镜像)      canonical_items (TMDb 匹配后)
    id ───────────────────────────→ external_id
                                   virtual_series_id ──→ virtual_series
                                                             ↑
                             同 TMDb tv_id 的多季聚合到这里
```

## 非目标

- ❌ 不填充 `virtual_series` 数据（属 P1.5-C）
- ❌ 不实现追踪逻辑（属 P1.5-D）
- ❌ 不删除 `baseline_items` 表或飞书相关旧字段（保持向后兼容）
- ❌ 不修改业务代码

## 输出

### 1. Migration 006 SQL

```sql
-- 006_virtual_series.sql

create table if not exists virtual_series (
    id integer primary key autoincrement,
    tmdb_tv_id text not null,
    name text not null,
    tmdb_status text,
    tmdb_number_of_seasons integer,
    local_max_season integer,
    poll_priority text not null default 'normal',
    last_polled_at text,
    created_at text not null default current_timestamp,
    updated_at text not null default current_timestamp
);

create unique index if not exists ux_virtual_series_tmdb_tv_id
on virtual_series(tmdb_tv_id);

-- canonical_items 新增 virtual_series_id（不设 FK 约束，低置信度可为 NULL）
alter table canonical_items add column virtual_series_id integer;

create index if not exists idx_canonical_items_virtual_series
on canonical_items(virtual_series_id);

-- content_updates 新增 match_confidence_low
alter table content_updates add column match_confidence_low integer default 0;

insert or ignore into schema_migrations(version) values (6);
```

### 2. schema.py 更新

- `SCHEMA_VERSION = 6`
- `initialize_database()` 中追加 `_apply_migration(conn, 6, ...)`
