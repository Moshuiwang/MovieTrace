## 来源任务

2026-05-13 追加决策：接受“降低 OMDb 依赖”；接受 TMDb 评分作为 IMDb 评分维度 fallback；P1.8-F / P1.8-G 合并执行，避免 external_ids client、缓存和统计重复实现。

## 目标
调整 `daily-discover` 主线顺序：在 OMDb enrichment 和 hot_score 计算前，对缺 IMDb ID 且有 TMDb ID 的候选即时调用 TMDb external_ids 补齐 IMDb ID；评分时若 IMDb rating/votes 仍缺失，则用 TMDb vote_average/vote_count 计算 IMDb 评分维度的 fallback 分，并在评分明细中标明来源。

本任务的产品方向是降低 OMDb 对当天评分链路的阻塞性：OMDb 有结果时优先使用真实 IMDb 分；OMDb 因配额、授权、缺 IMDb ID 或返回缺失导致不可用时，不阻断评分，改用 TMDb fallback。

## 非目标
- ❌ 不调整评分权重、P0/P1/P2 阈值或推荐排序规则。
- ❌ 不解决 OMDb key 配额、授权或 401 问题。
- ❌ 不补拉历史全量 external_ids。
- ❌ 不改变 TMDb popularity、TMDb rating 原有评分公式。
- ❌ 不把 TMDb fallback 分伪装成真实 IMDb 分。
- ❌ 不引入新依赖。
- ❌ 不提交、替换、打印或记录完整 API key / token。
- ❌ 不要求 OMDb 必须成功后才能给出 P0/P1/P2 评分。

## 具体要求

### 1. 主线顺序调整

`daily-discover` 的候选处理顺序应调整为：

1. 采集 FP / TMDb / Trakt 源数据。
2. 合并多源候选。
3. 对候选中 `imdb_id` 为空、`tmdb_id` 存在、`media_type` 为 `tv` 或 `movie` 的条目，立即查询 TMDb external_ids。
4. 将返回的 `imdb_id` 写回候选对象，并入库 external_ids 原始响应。
5. 执行 OMDb enrichment；如果 OMDb 因配额、授权、网络或响应缺失失败，记录失败并继续。
6. 执行 hot_score 计算和优先级映射；IMDb 评分维度按真实 IMDb 优先、TMDb fallback 次之。
7. 输出推荐结果和运行统计。

这一步不能只作为评分后的后置补拉，否则当天评分仍会受 IMDb 缺失影响。

### 2. TMDb external_ids 即时补齐规则

补齐对象：

- `candidate.imdb_id` 为空。
- `candidate.tmdb_id` 非空。
- `candidate.media_type` 可识别为 `tv` 或 `movie`。

跳过对象：

- 已有 `imdb_id`。
- 无 `tmdb_id`。
- `media_type` 为空、未知或不可映射。
- external_ids 缓存存在且确认 `imdb_id` 为空，可按缓存有效期跳过重复请求。

写回规则：

- TMDb 返回 `imdb_id` 时，写回候选。
- 同步写入 external_ids 入库结构或缓存。
- 如果已有其他来源 IMDb ID 与 TMDb 返回值冲突，不覆盖原值，只记录冲突。

### 3. OMDb enrichment 接入

TMDb external_ids 新补齐的 IMDb ID 必须进入同一次 `daily-discover` 的 OMDb enrichment。

验收时至少覆盖：

- 修复前：候选缺 IMDb ID，OMDb enrichment 被跳过。
- 修复后：候选通过 TMDb external_ids 得到 IMDb ID，并在同一次 run 中调用 OMDb enrichment。

如果 OMDb 因配额或授权失败无法返回评分，流程不得中断，应进入 TMDb 评分兜底。

### 4. IMDb 评分维度的 TMDb fallback

在综合评价阶段，如果以下任一条件成立：

- `imdb_id` 仍为空。
- `imdb_id` 已有，但 OMDb 未返回有效 `imdb_rating`。
- `imdb_votes` 为空或不可解析。

则使用 TMDb `vote_average` / `vote_count` 计算 IMDb 评分维度的 fallback 分。

约束：

- 不改变 `tmdb_rating_score` 原有计算。
- 不改变 `imdb_rating` / `imdb_votes` 原始字段含义。
- 不把 TMDb vote_average 写入 `imdb_rating` 字段。
- 在 `score_breakdown` 中增加可审阅字段，例如：
  - `imdb_rating_score_source`: `imdb` 或 `tmdb_fallback`
  - `imdb_rating_fallback_reason`: `missing_imdb_id` / `missing_omdb_rating` / `missing_omdb_votes`
  - `imdb_rating_score`: fallback 后实际用于该评分维度的分值

### 5. 评分公式要求

fallback 分值应复用现有 TMDb rating 计算逻辑，避免引入新公式：

- 使用 `compute_tmdb_rating_score` 对 TMDb `vote_average` / `vote_count` 计算 0-100 分。
- 将该分值填入 IMDb rating 维度参与 `imdb_rating` 权重计算。
- `score_breakdown` 必须能看出该分来自 TMDb fallback。

示例：

- 有真实 OMDb 评分：`imdb_rating_score_source = "imdb"`
- 无 OMDb 评分但有 TMDb vote：`imdb_rating_score_source = "tmdb_fallback"`
- 两者都没有：`imdb_rating_score_source = "missing"`，分值为 `None` 或 `0.0`，需沿用现有评分明细风格。

### 6. API 用量与限流

- external_ids 即时补齐请求必须可计数。
- 若 P1.8-D 已完成，写入 `api_usage_log`：
  - service=`tmdb`
  - endpoint=`external_ids`
  - operation=`pre_score_imdb_backfill`
  - http_status
  - cache_status
  - item_count
  - duration_ms
  - rate_limited
- 支持限制单次补齐请求量，避免当天候选过多时请求失控。
- 触发 429 时必须停止或退避，并在报告中记录。

### 7. 运行统计与报告

`daily-discover` 输出或报告中必须包含：

- imdb_backfill_scanned：扫描候选数
- imdb_backfill_eligible：可通过 TMDb external_ids 补齐的候选数
- imdb_backfill_cache_hit：external_ids 缓存命中数
- imdb_backfill_fetched：实际 TMDb 请求数
- imdb_backfill_filled：新补 IMDb ID 数
- imdb_backfill_conflicts：IMDb ID 冲突数
- imdb_score_real：使用真实 OMDb IMDb 评分数
- imdb_score_tmdb_fallback：使用 TMDb 评分兜底数
- imdb_score_missing：IMDb 和 TMDb 评分均不可用数

## 验收标准
1. ✅ `daily-discover` 在 OMDb enrichment 和评分前执行 IMDb ID 补齐。
2. ✅ 缺 IMDb ID 但有 TMDb ID 的候选可通过 TMDb external_ids 在同一次 run 中补齐。
3. ✅ 新补 IMDb ID 会进入同一次 OMDb enrichment。

6. ✅ 不把 TMDb vote_average 写入 `imdb_rating` 字段。
7. ✅ external_ids 请求可统计、可限流、不泄露 token。

## 风险点
- TMDb fallback 会让 TMDb rating 同时影响 `tmdb_rating` 和 `imdb_rating` 两个评分维度；这是本任务的明确产品取舍，必须在 breakdown 中透明标记。
- 如果 TMDb external_ids 请求量不受控，daily run 会增加明显 API 调用量。
- 如果 OMDb key 当天已达配额，新增 IMDb ID 仍无法换来真实 IMDb 分；这是已接受的产品取舍，必须进入 TMDb fallback 并记录原因。
- 如果候选 media_type 归一化不稳定，可能导致 external_ids 路径选错，需要测试覆盖 `tv` / `movie` / `show` 映射。
- fallback 后 P2+ 数量可能上升，验收报告需要对比修复前后分布。
