## 来源任务
来自 FROM 缺失排查与 FlixPatrol API 用量讨论：当前只抓美国区 6 平台 Movie/TV 日榜，无法覆盖 FROM 在全球榜中的热度；同时平台/国家盲目扩大会超过月度 API 预算，因此需要明确 FP 抓取国家、平台和频率策略。

## 目标
将 FlixPatrol 抓取策略调整为：4 个国家/范围、6 个平台、TV 榜每日抓取、Movie 榜每周抓取一次，并在运行统计中可估算月度 API 消耗。

- ❌ 不抓 Overall TOP10、Kids TOP10、周榜/月榜/年榜 rankings。
- ❌ 不抓平台不限的全量 FP 平台。
- ❌ 不补拉历史全量 FP 数据。
- ❌ 不改变 FlixPatrol 每 URL 每 24h ≤ 1 次、间隔 ≥ 2 秒、UA = `MovieTraceBot/0.1` 的合规要求。
- ❌ 不提交、替换、打印或记录完整 API key。

## 具体要求

### 1. 国家/范围策略

FP 抓取范围改为以下 4 个：

- World
- United States
- Nigeria
- Kenya

要求：

- 不再只硬编码美国。
- 国家/范围 ID 必须集中定义，优先放在配置或清晰常量中。
- 如果 World 在 FP API 中不是普通 country ID，必须通过最小请求验证并在报告中说明实现方式。
- 国家名称写入 `flixpatrol_top10.country` 时必须可区分，例如 `world`、`united-states`、`nigeria`、`kenya`。

### 2. 平台策略

FP 平台限制为以下 6 个：

- Netflix
- Prime Video
- Disney+
- Apple TV+
- HBO Max
- Paramount+

要求：

- 移除 Hulu。
- 增加 Paramount+。
- 平台 ID 必须集中定义。
- `COMPANY_TO_PLATFORM` 映射必须同步更新。
- 平台名称写入 `flixpatrol_top10.platform` 时必须稳定，例如 `paramount-plus`。

### 3. 榜单类型与频率

只抓两类榜单：

- TV Shows TOP10：每日抓取。
- Movies TOP10：每周抓取一次。

调度要求：

- `daily-discover` 每天运行时必须抓 TV。
- Movie 抓取只在配置的周抓取日执行，默认建议为每周一。
- 应支持通过 CLI 参数或内部选项强制抓 Movie，便于人工验证。
- 非 Movie 抓取日不得请求 Movie TOP10 API。
- 不抓 Overall / Kids。

### 4. API 调用预算

每次 FP 抓取前应能计算计划请求数：

- TV daily 请求数：`国家/范围数 × 平台数 × 1`
- Movie weekly 请求数：`国家/范围数 × 平台数 × 1`

按当前策略：

- 普通日仅 TV：`4 × 6 = 24` 次。
- Movie 抓取日 TV + Movie：`4 × 6 × 2 = 48` 次。
- 30 天月估算：约 `824` 次。
- 31 天 + 5 个 Movie 抓取周估算：约 `864` 次。

运行统计或日志中必须输出：

- fp_planned_calls
- fp_actual_calls
- fp_tv_calls
- fp_movie_calls
- fp_country_count
- fp_platform_count
- fp_monthly_estimate_low
- fp_monthly_estimate_high

### 5. 合规与限流

- 每个 FP URL 每 24h 不重复请求。
- 请求间隔必须 ≥ 2 秒。
- User-Agent 必须保持 `MovieTraceBot/0.1`。
- 遇到 429 不得无限重试。
- 不输出完整 API key。

### 6. 入库行为

- `flixpatrol_top10` 继续按榜单行入库。
- `snapshot_date` 必须等于请求日期。
- `country` 必须反映对应国家/范围。
- 同一日期、平台、国家、内容类型、排名的记录不得重复写入。
- 旧的 Hulu 历史数据不删除；新运行不再产生 Hulu 数据。

## 验收标准
1. ✅ FP 国家/范围为 World、United States、Nigeria、Kenya。
2. ✅ FP 平台为 Netflix、Prime Video、Disney+、Apple TV+、HBO Max、Paramount+，且不再抓 Hulu。
3. ✅ TV Shows TOP10 每日抓取。
4. ✅ Movies TOP10 默认每周抓取一次，非抓取日不产生 Movie API 请求。
5. ✅ 普通日计划 FP 请求数为 24，Movie 抓取日计划请求数为 48。
6. ✅ 月度 API 估算低值约 824、高值约 864，并低于 1000 calls/month。
7. ✅ FP 合规限制仍满足：每 URL 每 24h ≤ 1 次、间隔 ≥ 2 秒、UA 正确。
8. ✅ 全量测试通过。

## 风险点
- World、Nigeria、Kenya、Paramount+ 的 FP API ID 需要通过官方接口或最小请求确认；不能凭页面文案硬猜。
- 如果 FP 套餐月度上限严格按自然月重置，连续 31 天且 5 个 Movie 抓取周仍有余量，但需要避免人工重复 dry-run 消耗过多 API。
- Paramount+ 在不同国家/范围可能没有 TV 或 Movie 日榜，空结果应被视为正常可审查状态，不应静默当作失败。
- Movie 每周抓取会降低电影榜时效性，这是本任务接受的 API 预算取舍。
- 增加 World 和非美国国家后，FP 信号可能显著改变 P2+ 分布，报告中需要对比修改前后候选数量和 FROM 是否获得 FP 信号。
