# 任务包：P1.12-F — Secrets 路径迁移到 ~/.config/movietrace/

**任务包版本：** v1
**创建时间：** 2026-05-14 15:30 +08

## 任务名称
P1.12-F：Secrets 路径从 `/tmp` 迁移到 `~/.config/movietrace/secrets.json`

## 任务类型
fix

## 当前阶段
Phase 1.12：review hotfix

## 来源任务
CR-007（`reports/code_review_2026-05-14.md`）+ ADR-0011 决策确认。

## 目标
将 secrets 默认路径从 `/tmp/movietrace_phase0_secrets.json` 迁移到 `~/.config/movietrace/secrets.json`，收敛读取逻辑到单一配置模块，消除 8 处硬编码。

## 非目标
- ❌ 不改变 secrets JSON 内部格式。
- ❌ 不改 API key 本身的值。
- ❌ 不做 secrets 加密。
- ❌ 不影响 P1.12-A~E 其他 hotfix。

## 允许修改范围
- `src/movietrace/config.py`（新建：`load_secrets()` + `get_secrets_path()` + 权限检查）
- `src/movietrace/cli.py`（替换 5 处硬编码）
- `src/movietrace/sources/flixpatrol_api.py`（替换模块级常量）
- `src/movietrace/pipeline/discovery.py`（替换 `_load_secrets()`）
- `src/movietrace/pipeline/entity_matching.py`（替换 --secrets default）
- `tests/test_config.py`（新建）
- `tests/test_cli.py`、`tests/test_discovery.py`、`tests/test_flixpatrol_api.py`（适配新路径）

## 禁止修改范围
- 🚫 `/tmp/movietrace_phase0_secrets.json` 内容
- 🚫 AGENTS.md、CLAUDE.md
- 🚫 DB schema / migrations
- 🚫 外部 API client 行为

## 相关上下文
- 8 处硬编码分布在 4 个文件：`cli.py`(5) · `flixpatrol_api.py`(1) · `discovery.py`(1) · `entity_matching.py`(1)
- ADR-0011 已确认迁移方向，详见 `docs/decisions/0011-secrets-path-migration.md`
- 当前 `_load_secrets()` 在 `cli.py` 和 `discovery.py` 各有一份重复实现

## 输入
- ADR-0011 设计决策
- 当前硬编码路径清单

## 输出
- `src/movietrace/config.py` 模块（单一 secrets 读取入口）
- 4 个文件 8 处硬编码替换为 `config.load_secrets()` 调用
- 向后兼容：新路径不存在时 fallback 旧路径 + deprecation warning
- 权限检查：非 0600 时 warning

## 具体要求
1. 新建 `src/movietrace/config.py`：
   ```python
   from pathlib import Path
   import os
   import json
   import logging

   logger = logging.getLogger(__name__)

   DEFAULT_SECRETS_DIR = Path.home() / ".config" / "movietrace"
   DEFAULT_SECRETS_PATH = DEFAULT_SECRETS_DIR / "secrets.json"
   LEGACY_SECRETS_PATH = Path("/tmp/movietrace_phase0_secrets.json")

   def get_secrets_path() -> Path:
       """返回优先使用的 secrets 路径。"""
       ...

   def load_secrets(path: str | Path | None = None) -> dict:
       """加载 secrets，新路径优先，fallback 旧路径并 warn。"""
       ...

   def _check_permissions(path: Path) -> None:
       """检查文件权限是否为 0600，否则 warning。"""
       ...
   ```
2. 替换全部 8 处硬编码。
3. `discovery.py::_load_secrets()` 删除，改为 `from movietrace.config import load_secrets`。
4. `cli.py::_load_secrets()` 删除，改为 `from movietrace.config import load_secrets`。
5. 向后兼容：`load_secrets()` 先尝试 `~/.config/movietrace/secrets.json`，不存在时读 `/tmp/movietrace_phase0_secrets.json` 并输出 deprecation warning。
6. 测试中 mock `DEFAULT_SECRETS_PATH` 指向临时文件，不触碰真实 secrets。

## 验收标准
1. ✅ `config.load_secrets()` 可从新路径正常加载。
2. ✅ 新路径不存在时 fallback 旧路径 + 输出 warning。
3. ✅ 8 处硬编码全部替换为统一入口。
4. ✅ `discovery.py` 和 `cli.py` 不再有重复的 `_load_secrets()`。
5. ✅ 全量测试通过（mock secrets path，不依赖真实文件）。

## 测试要求

- 现有测试适配新 import 路径

- 切换后旧路径被系统清理会导致 fallback 失败，用户需手动迁移
- 测试中的 secrets path mock 需要确保不触碰真实文件
