# 任务包：P1.12-C — OMDb key 日志脱敏

**任务包版本：** v1
**创建时间：** 2026-05-14 15:05 +08

## 任务名称
P1.12-C：OMDb key 日志脱敏，禁止输出完整短 key

## 任务类型
fix

## 当前阶段
Phase 1.12：review hotfix

## 来源任务
2026-05-14 Codex 全面代码 review：`enrich_with_omdb()` 在 key 失效时 logger 输出 `key[:8]`，而 OMDb key 常见长度正好为 8，存在完整 key 泄露风险。

## 目标
所有 OMDb key 相关运行日志只输出不可逆 fingerprint 或安全 mask，绝不输出完整 key。

## 非目标
- ❌ 不修改 secrets 文件格式。
- ❌ 不修改 `api_usage_log.key_fingerprint` 设计。
- ❌ 不扩展到 Feishu/TMDb/Trakt 的全部日志审计。
- ❌ 不做新集更新追踪。

## 允许修改范围
- `src/movietrace/pipeline/omdb_enrichment.py`
- `src/movietrace/logging/api_usage.py`（仅如复用/新增 mask helper 必需）
- `tests/test_omdb_enrichment.py`
- `tests/test_api_usage_logging.py`（仅如新增 helper）

## 禁止修改范围
- 🚫 AGENTS.md、CLAUDE.md
- 🚫 `/tmp/movietrace_phase0_secrets.json`
- 🚫 `.env`
- 🚫 DB schema / migrations
- 🚫 OMDb 多 key 轮转语义

## 相关上下文
当前风险点：

```python
logger.warning("OMDb key %s... HTTP %s", key[:8], exc.status_code)
```

如果 key 为 `c9c22b79` 这类 8 位值，日志会包含完整 key。

## 输入
- OMDb 多 key 轮转逻辑
- `fingerprint_key()` helper

## 输出
- OMDb key 日志显示类似 `fp=abcd1234ef56` 或 `c9***79`
- 单测证明日志内容不包含完整 key

## 具体要求
1. 替换 `key[:8]` 输出方式。
2. 优先使用不可逆 fingerprint：`fingerprint_key(key)`。
3. 如使用 mask，短 key 也必须至少隐藏中间和尾部足够字符，不能完整出现。
4. 新增日志捕获测试：模拟 FatalApiError，断言完整 key 不在日志文本中。
5. 保持返回 stats 不变。

## 验收标准
1. ✅ 8 位 OMDb key 不会完整出现在 logger 输出。
2. ✅ 多 key 轮转行为不退化。
3. ✅ API usage logging 原有脱敏测试仍通过。
4. ✅ 全量测试通过。

## 测试要求
- 在 `tests/test_omdb_enrichment.py` 增加 `assertLogs` 或 caplog 等价测试。
- 覆盖短 key 和长 key 至少一种场景。

## 验证命令
```bash
PYTHONPATH=src .venv/bin/python -m pytest tests/test_omdb_enrichment.py tests/test_api_usage_logging.py -v
PYTHONPATH=src .venv/bin/python -m pytest tests/ -v
```

## 风险点
- 日志可观测性会从“key 前缀”改为 fingerprint，需要运维排查时用 fingerprint 对照 key。
- 不应在测试 failure message 中输出完整 key。
