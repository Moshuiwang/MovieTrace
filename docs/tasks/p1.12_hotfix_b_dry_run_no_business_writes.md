## 来源任务
2026-05-14 Codex 全面代码 review：`run_discovery(dry_run=True)` 仍会执行 auto-register，写入 `canonical_items` / `external_ids`。

## 目标
修复 `run_discovery(dry_run=True)` 的写库语义：dry-run 可读取、抓取、计算和返回结果，但不得写入业务结果表 `canonical_items`、`external_ids`、`content_updates`。

- ❌ 不改变源数据缓存/抓取状态/用量日志的现有 dry-run 约定。
- ❌ 不改变评分、阈值或推荐结果。
- ❌ 不实现全局事务回滚框架。
- ❌ 不做新集更新追踪。

## 允许修改范围
- `src/movietrace/pipeline/discovery.py`
- `src/movietrace/cli.py`（仅如需输出 dry-run auto-register 统计）
- `tests/test_discovery.py`
- `tests/integration/test_daily_discover_pipeline.py`

## 禁止修改范围
- 🚫 AGENTS.md、CLAUDE.md
- 🚫 DB schema / migrations
- 🚫 `config/scoring_weights.yaml`
- 🚫 外部 API client 行为

## 相关上下文
当前复现：

```json
{"before_canonical": 0, "after_canonical": 1, "after_external_ids": 1, "auto_registered": 1, "passed": 1}
```

原因是 auto-register 逻辑位于 `if not dry_run: _write_content_updates(...)` 之前。

## 输入
- `run_discovery(date_from=..., dry_run=True, db_path=...)`
- 包含 P2+ candidate 且无 canonical 映射的临时 DB

## 输出
- dry-run 结果中可以报告 `auto_register_candidates` 或类似统计
- dry-run 不新增 `canonical_items` / `external_ids` / `content_updates`
- 非 dry-run 仍能 auto-register 并写 content_updates

## 具体要求
1. `dry_run=True` 时不得调用 `_ensure_canonical_item()`。
2. dry-run 可返回将会注册的数量，但字段名必须清晰区分“实际写入”和“预估写入”。
3. `dry_run=False` 行为保持：P2+ 且无 canonical 的 candidate 自动注册后写入 content_updates。
4. 增加回归测试覆盖临时 DB 的表计数。
5. 保持 P1.10 fallback/source status 当前行为不在本任务内调整。

## 验收标准
1. ✅ dry-run 前后 `canonical_items` 行数不变。
2. ✅ dry-run 前后 `external_ids` 行数不变。
3. ✅ dry-run 前后 `content_updates` 行数不变。
4. ✅ 非 dry-run 仍能写入必要业务结果。
5. ✅ 全量测试通过。
