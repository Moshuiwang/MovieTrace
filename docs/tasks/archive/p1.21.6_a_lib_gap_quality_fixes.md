## 具体要求

### 1. 虚假缺口剔除（HIGH）

修改 `src/movietrace/feishu/gap_sync.py` 的 `_GAP_SQL`：

WHERE 子句增加：
```sql
AND COALESCE(alm.a_lib_max_season, 0) > 0
```

效果：A 库 中**完全未收录的剧**不再以"缺口"形式出现在 A 库缺口表。这些剧应通过热点发现表（new_discovery）发现，属于不同语义层。

### 2. 追上行删除（Option A）

`sync_gap_table` 新增 step 6：

- 计算 `tmdb_ids_in_sql = {row["tmdb_tv_id"] for row in rows}`
- `tmdb_ids_in_feishu = set(existing_map.keys())`
- `caught_up_ids = tmdb_ids_in_feishu - tmdb_ids_in_sql`
- `record_ids_to_delete = [existing_map[tid] for tid in caught_up_ids]`
- 调用 `_batch_delete_records(token, app_token, table_id, record_ids_to_delete)`
- 输出统计：`{deleted: N}`

新增 `_batch_delete_records` helper（参考 `scripts/feishu_cleanup.py` 中的实现，但放到 `_http.py` 共享）：
```python
def batch_delete_records(token, app_token, table_id, record_ids):
    url = f"{OPEN_API_BASE}/bitable/v1/apps/{app_token}/tables/{table_id}/records/batch_delete"
    for chunk in chunks(record_ids, 500):
        resp = request_json("POST", url, token=token, payload={"records": chunk})
        if resp.get("code") != 0: raise RuntimeError(...)
```

### 3. 字段说明子表（MEDIUM）

通过 API 向 `tblPXLrWEEf4bhtM` 写入新行（或更新现有行）。字段：字段名 / 含义 / 备注。最少补：

- "季号"：TMDb 当前已播总季数（非"这一季是第几季"）
- "A 库 最新季"：A 库 数据库当前收录的该剧最大 season_number；"无" 表示 A 库 完全未收录此剧
- "TMDb 已播季"：TMDb `last_episode_to_air.season_number`（不含已宣布未上映季）
- "缺口数"：TMDb 已播季 − A 库 当前最大季
- "运营状态" 各选项含义（待补 / 部分补充 / 已补 / 跳过）
- "数据源状态"：当日 FlixPatrol / TMDb / Trakt 的抓取结果（fresh / fallback / failed）

### 4. 热点发现类型字段选项整理

通过 PUT `/bitable/v1/apps/{app_token}/tables/{table_id}/fields/{field_id}` 更新 `fldeHJXf1a`：

当前选项：`['TV', 'Movie', 'tv', 'movie']`
目标选项：`['tv', 'movie']`

**注意**：先确认现有数据 0 行使用大写选项再删除（已确认：data 全部小写）。如果删大写选项导致旧数据指针失效，会被 Feishu 拒绝；备用方案：保留大写选项作为 deprecated，仅文档说明。

### 5. A 库缺口排序视图

通过 POST `/bitable/v1/apps/{app_token}/tables/{table_id}/views`（如果飞书 API 支持）创建：
- 视图名："待补 - 在播中"
- filter: `TMDb 状态 = Returning Series` AND `运营状态 NOT IN (已补, 跳过)`
- sort: `hot_score DESC, 缺口数 DESC`

如果 API 不支持自动建视图，记录到任务报告中，让用户手工建。
