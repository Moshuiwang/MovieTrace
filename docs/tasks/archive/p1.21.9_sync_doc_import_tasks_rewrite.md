## 具体要求

### 1. 三步流实现

按飞书官方 import_task 文档执行（Agent 用 WebFetch 拉取实时参数细节，**不要硬编码本任务包列出的字段名作为唯一来源**）：

**步骤 A — 上传 md 文件**
- 端点（参考）：`POST /open-apis/drive/v1/medias/upload_all`
- 类型：multipart/form-data
- 文件名必须严格 `.md` 后缀（否则飞书可能报 1069910 不匹配错误）
- 返回 `file_token`

**步骤 B — 创建导入任务**
- 端点（参考）：`POST /open-apis/drive/v1/import_task/create`
- JSON body：包含 `file_extension="md"`、`file_token`、目标类型 `type="docx"`、`file_name`、挂载点（mount_type / mount_key）
- 返回 `ticket`

**步骤 C — 轮询任务结果**
- 端点（参考）：`GET /open-apis/drive/v1/import_task/{ticket}` 或 `GET /open-apis/drive/v1/import_task/get/{ticket}`（以官方文档为准）
- 轮询策略：**指数退避** delays = [1, 2, 4, 8, 16, 30, 30, 30, 30, 30] 秒（累计约 3 分钟），**硬超时 5 分钟**
- 成功后从响应中提取 `url` 和 `token`（具体字段路径以官方文档为准）

### 2. 错误处理与失败兜底

- 非业务 HTTP 错误（5xx / timeout）：raise RuntimeError，由 `cmd_sync_feishu_doc` 的 try/except 捕获并返回 1
- permission denied 类错误码（99991661 / 99991663 / 1061045 / 飞书文档列出的"权限不足"码）：抛 RuntimeError 时**显式提示"在飞书控制台为 app 申请 drive 相关 scope"**
- 轮询超时（达到 5 min 仍非终态）：raise RuntimeError，附 ticket 便于 debug
- dry_run 模式：仅读 md 文件确认存在、打印长度和 title，**不调用任何飞书接口**，返回 `{"dry_run": True}`

### 3. multipart 上传实现要求

stdlib only（不引入 `requests` / `httpx`）：
- 构造 `multipart/form-data` body：随机 boundary（用 `secrets.token_hex(16)`，确保未在文件内容中出现）
- 每个 part 含 `Content-Disposition` 头；文件 part 含 `Content-Type: application/octet-stream`
- 严格 RFC 7578 的 `\r\n` 分隔
- 通过 `urllib.request.Request` 发送，header `Content-Type: multipart/form-data; boundary=<boundary>`
- multipart body 构造逻辑**必须独立成函数并单测**（输入 dict + bytes → 输出 bytes + boundary str）

### 4. 删除旧实现

- 删除 `_DOCX_BLOCK_MAX_CHARS` 常量
- 删除原 sync_doc 内 `POST /docx/v1/documents` 与 `POST /docx/v1/documents/{id}/blocks/{id}/children` 的所有代码
- 函数 docstring 更新：说明走 import_task 路径、扩展名约束、所需 scope

### 5. ADR-0014 内容要求

`docs/decisions/0014-feishu-doc-import-via-import-tasks.md`，≤200 行，覆盖：
- **背景**：为什么 plain text docx 不够用
- **三方案对比**：A 当前 docx blocks / B blocks/convert 同步 / C import_task 异步
- **选择 C 的理由**：保真度、稳定性、官方推荐
- **已知 caveats**：异步轮询、扩展名严格、新 scope 申请、轮询超时偶发误判
- **回滚策略**：如果失败率高，可回退到 B（blocks/convert 同步方案）；保留 B 的实现路径草图

## 风险点

- **飞书 scope 申请非代码层面**：用户必须在飞书控制台手动申请 `drive` 相关 scope，Agent 不能代办。如果跑到上传/导入步骤返回权限错误，Agent **必须停止并报告**，不要绕过。
- **import_task 异步轮询的时长**：md 文件不大（reports/latest.md 通常 <100KB），实际导入应在 5-30 秒；但飞书后端忙时可能拖到分钟级，5 min 硬上限可能偶发误判，可观察一段后调整。
- **multipart body 手写细节**：boundary 必须未在文件内出现，`\r\n` 严格遵守 RFC 7578；任何一处错都会被飞书拒。
- **mount_point 字段在不同 API 版本可能不同**：执行 Agent **必须以 WebFetch 拉取的官方文档实时字段为准**，不要硬编码本任务包写的猜测字段名。
- **回退策略**：如果上线后失败率高，回退方案是 blocks/convert（同步），ADR 必须写明回退路径。
