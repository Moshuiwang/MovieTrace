### 周报字段定义（observability 指标）

每份周报必须包含以下数据切片：

**A. 基本信息**
- 周编号（ISO 周）、起止日期、生成时间、数据拉取时间

**B. 热点发现表统计（覆盖最近 7 天）**
- 总候选数（按发现日期去重的行数）
- 按 `运营状态` 分布：待看 / 确认加入 / 不加入 / 空
- 按 `优先级` × `运营状态` 交叉表（看 P0 是不是更容易被确认）
- 运营回填率 = (有运营状态的行数) / 总数
- 采纳率 = (确认加入) / (确认加入 + 不加入)（分母为空时显示 N/A）
- 供应商推进：未提交 / 已提交 / 有货 / 无货 分布

**C. A 库缺口表统计（当前快照）**
- 总 series 数（有缺口的）
- 按状态分布：待补 / 部分补充 / 已补 / 跳过 / 空
- 推进率 = (已补 + 跳过) / 总数
- Top 10 高 hot_score 待补 series（标题 + 缺口季 + hot_score）

**D. 关键案例**
- 本周被标"不加入"的 P0/P1 案例（最多 10 条；标题 + hot_score + 运营备注）——这是误报信号源
- 本周被标"确认加入"但 hot_score < 60 的案例——这是评分公式可能低估的信号

**E. V2 触发条件检查（沿用现有模板字段）**
- V1 稳定运行 1-2 月：累计天数
- 运营反馈具体需求短板：本周新备注数 + 摘录
- 其他两项保持"否"，等积累后人工评估

## 具体要求

### 1. 拉取行为
- 分页：飞书 Bitable list/search records 单页上限 500，必须实现分页直到结束
- 字段 ID 引用而非名称（保护字段改名场景）
- 拉取失败：单次 HTTP 错误重试 2 次（指数退避 1s / 2s），3 次失败则整体失败并返回非零退出码
- 不缓存运营字段历史（每周快照即周报，历史靠 git）

### 2. 周报生成
- 周编号：`datetime.isocalendar().week`，文件名 `feedback_log_YYYY-Www.md`（`W` 大写两位数）
- 如果当前 ISO 周已有 md 文件，覆盖（同周内多次跑就是刷新；跨周保留历史）
- 标题、TMDb ID 等需要从 B 库 SQLite 反查时，找不到的不报错，留 `?` 占位

### 3. CLI 参数
- `pull-feishu-feedback --days 7 --output reports/feedback/`（默认 7 天、默认目录）
- `export-feedback-report --input reports/feedback/feishu_pull_latest.json --output reports/feedback/`
- `--dry-run`：不写文件，只 print 摘要

### 4. 飞书 spec 文档（最低门槛）
`docs/operations/feishu_feedback_spec.md` 至少包含：
- 运营每天/每周需要看哪张表、看什么
- "确认加入" vs "不加入" 的判断口径（如：是否对非洲英文用户有市场 / 是否已有同类）
- "待补 / 已补 / 跳过" 的判断口径
- 不填会怎样（明确不影响系统行为，只影响周报含金量）

### 5. 不接 cron
- `scripts/weekly_feedback.sh` 写好但 **不**写入 `crontab -l`
- 任务包内不修改 cron 配置文件
- README/runbook 注明"周日手动跑"

## 风险点

- **R1：W0 周报会很空**。当前飞书运营字段几乎全部为空，第一份周报采纳率会显示 N/A、回填率接近 0。这是**预期行为，不是 bug**；周报本身就是为了让你看到"运营是不是真的在填"。
- **R2：飞书字段在 P1.21 后是否被改名/改字段类型未知**。任务内通过字段 ID（不是名称）引用，但如果运营/你自行改了 select 选项值（如把"确认加入"改成"采纳"），周报里的分类会失配。需要在 `feishu_feedback_spec.md` 里明确"不要改 select 选项值，只填，不改字段配置"。
- **R3：拉取范围 7 天 vs 飞书表实际只追加新行**。热点发现表 sync-feishu-table 当天 upsert，跨天 append，所以"近 7 天"用 `发现日期` 字段过滤而不是 `同步时间`。
- **R4：周观察"还不够"**。SCOPE 触发 V2 的硬条件是 1-2 个月，本任务只是建好通道，**不代表观察期完成**。任务包完成后，下一阶段决策（是否启动 V2、是否调整评分、是否回写）仍需要后续多周积累。
