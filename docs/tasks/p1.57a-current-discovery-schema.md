## 来源任务
[ADR-0016](../decisions/0016-current-discovery-with-observations.md) 决定将 `daily-discover` discovery 输出从每日事件行改为当前发现项 + observation 留痕。

## 目标
新增 B 库 schema，使系统能存储“一个稳定内容一行”的当前发现项，以及“一个有效观察日一行”的 observation 留痕。

## 非目标
- ❌ 不改 `daily-discover` 写入逻辑
- ❌ 不改导出、飞书同步或通知
- ❌ 不迁移历史 `content_updates` 数据
- ❌ 不新增 `score_version` / `scoring_config_hash`
- ❌ 不改变 `new_season` 的 `content_updates` 事件历史语义

## 允许修改范围

## 输出
- `current_discovery_items` 表
- `discovery_observations` 表
- 必要唯一索引和查询索引
- schema / migration 回归测试

## 具体要求
1. `current_discovery_items.discovery_key` 使用稳定键：`discovery:{movie|tv}:{tmdb_id}`，唯一且不含日期。
2. 当前发现项至少保存：首次发现日期、最近发现日期、发现次数、最新 `hot_score`、最新优先级、最新来源摘要、稳定 metadata、时间戳。
3. `discovery_observations` 至少保存：`discovery_key`、`observed_date`、当天来源摘要、原始热度输入或等价 JSON、score breakdown、当时 `hot_score`、当时优先级、时间戳。
4. `discovery_observations` 对 `(discovery_key, observed_date)` 加唯一约束，防止同一天重复 observation。
5. 不新增评分版本字段；历史分数只作为当时规则快照。
6. migration 必须可重复执行且不破坏已有数据。
7. migration 不得依赖当前开发库里的已知行数、已知标题、已知飞书记录或任何一次性人工信息；未来生产只应依赖旧 DB 文件本身完成结构升级。
