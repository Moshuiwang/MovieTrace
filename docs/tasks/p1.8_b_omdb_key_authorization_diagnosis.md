## 具体要求
1. 只读解析 `/tmp/movietrace_phase0_secrets.json`，确认：
   - 文件是否存在
   - JSON 是否有效
   - `omdb.api_key` 是否存在
   - key 是否为空字符串
   - 只允许输出脱敏信息，例如首 4 位 + `***`，禁止输出完整 key
2. 用最小请求验证 OMDb：
   - 使用一个稳定 IMDb ID，例如 `tt0111161`
   - 调用现有 `OmdbDetailClient.get_by_imdb_id()` 或等价最小 HTTP 请求
   - 记录 HTTP 状态、返回体中的 `Response` / `Error` 字段
   - 禁止批量请求
3. 检查调用格式：
   - 确认请求参数是否为 OMDb API 预期的 `apikey=<key>&i=<imdb_id>`
   - 确认 `format_imdb_id()` 对纯数字和 `tt` 前缀 ID 的行为
4. 检查缓存影响：
   - 查询 `api_cache` 中 `source='omdb'` 的条目数量
   - 抽样确认缓存 key 格式为 `omdb:tt...`
   - 不删除、不刷新缓存
5. 输出明确结论：
   - key 有效
   - key 缺失
   - key 无效 / 未授权
   - 配额耗尽
   - 调用格式错误
   - 网络 / 服务异常
   - 如无法判断，写明已排除项和下一步人工动作

## 风险点
- OMDb 免费 key 可能有每日配额限制；单次验证应尽量只发 1 个请求。
- OMDb 返回 `401` 可能由 key 无效、未激活、配额、IP/服务策略或调用格式引起，需结合响应体判断。
- 不能把真实 key 写入报告、日志、git diff 或对话输出。
