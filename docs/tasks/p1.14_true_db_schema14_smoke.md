- `src/movietrace/db/schema.py` 当前 `SCHEMA_VERSION = 14`。
- Migration 013：规范化 TMDb external_ids 命名空间。
- Migration 014：将 `content_updates` 改为事件历史表，以 `content_update_id` 唯一。
- 2026-05-14 预检查结果：真实库无重复 `content_update_id`；存在 1 条 legacy discovery id 和 420 条裸 TMDb external_id，属于 migration 013/014 预期处理范围。

## 具体要求
1. 执行前必须备份真实库，备份文件名带分钟级时间戳。
2. 执行前记录当前 schema version、`content_updates` 相关索引、重复 `content_update_id` 检查结果。
3. 用既有 `initialize_database('data/movietrace.db')` 执行 migration，不手写临时 SQL 改库。
4. 执行后确认：
   - `schema_migrations` 最大版本为 14。
   - `ux_content_updates_item_type` 不存在。
   - `ux_content_updates_update_id` 存在。
   - 裸 TMDb external_id 数量为 0。
   - legacy discovery id 数量为 0。
5. 跑完整测试，确认代码层回归仍通过。
6. 跑 `daily-discover --dry-run`，确认当前配置和真实库可完成最小流程。
7. 跑 `inspect-updates --days 7` 与 `export-recommendations --days 7`，确认最近事件可查询和导出。
8. 如任一验证失败，停止后续步骤，报告失败现象、已排除内容和下一步定位计划。

## 验收标准
1. ✅ 真实库已备份。
2. ✅ `data/movietrace.db` schema version 为 14。
3. ✅ `content_updates` 使用 `content_update_id` 唯一索引，不再使用 `(canonical_item_id, update_type)` 唯一索引。
4. ✅ 全量测试通过。
5. ✅ dry-run 与 inspect/export smoke 验收完成并记录关键输出。

- 必须运行全量 pytest。
- 必须执行真实库 dry-run smoke。
- 不要求新增测试文件。

## 验证命令
```bash
cp data/movietrace.db data/movietrace_backup_$(date +%Y%m%d_%H%M).db
sqlite3 data/movietrace.db "select max(version) from schema_migrations;"
sqlite3 data/movietrace.db "select content_update_id, count(*) c from content_updates group by content_update_id having c > 1 limit 20;"
PYTHONPATH=src .venv/bin/python -c "from movietrace.db.schema import initialize_database; initialize_database('data/movietrace.db')"
sqlite3 data/movietrace.db "select max(version) from schema_migrations;"
sqlite3 data/movietrace.db "select name from sqlite_master where type='index' and name in ('ux_content_updates_item_type','ux_content_updates_update_id') order by name;"
sqlite3 data/movietrace.db "select count(*) from content_updates where content_update_id glob 'discovery:[0-9]*:*';"
sqlite3 data/movietrace.db "select count(*) from external_ids where source='tmdb' and external_id not like 'movie:%' and external_id not like 'tv:%' and external_id not like 'unknown:%';"
PYTHONPATH=src .venv/bin/python -m pytest tests/ -v
PYTHONPATH=src .venv/bin/python -m movietrace.cli daily-discover --dry-run
PYTHONPATH=src .venv/bin/python -m movietrace.cli inspect-updates --days 7
PYTHONPATH=src .venv/bin/python -m movietrace.cli export-recommendations --days 7
```

## 风险点
- 真实库 migration 是不可逆写操作，必须先备份。
- `daily-discover --dry-run` 仍可能触达外部 API；如 API 授权、配额或网络失败，应如实记录，不得声明完整 smoke 通过。
- FP API 当前 402 是已知外部阻塞，不应在本任务中修复。
- `export-recommendations` 可能生成新报告文件；这是验证产物，不代表业务数据写入。
