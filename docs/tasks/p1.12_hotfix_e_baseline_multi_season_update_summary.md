# 任务包：P1.12-E — 多新季 content_update 汇总修复

**任务包版本：** v1
**创建时间：** 2026-05-14 15:05 +08

## 任务名称
P1.12-E：同一剧一次检测到多个新季时，content_update 不丢失季号信息

## 任务类型
fix

## 当前阶段
Phase 1.12：review hotfix

## 来源任务
2026-05-14 Codex 全面代码 review：`detect_new_seasons()` 可为同一 virtual_series 生成 S2/S3 多个事件，但 `content_updates` 唯一索引 `(canonical_item_id, update_type)` 导致只保留第一条。

## 目标
在不改 schema 的前提下，将同一剧同一次检测到的多个新季汇总为一条 `new_season` content_update，并在 `source_summary_json` 中保留完整季号列表/范围。

## 非目标
- ❌ 不重新设计 `content_updates` 唯一键。
- ❌ 不新增 `new_episode` / 新集更新追踪。
- ❌ 不回填历史 content_updates。
- ❌ 不改变 poll scheduler。

## 允许修改范围
- `src/movietrace/pipeline/baseline_tracking.py`
- `src/movietrace/reports/export_writer.py`（仅如需展示季号范围）
- `src/movietrace/reports/inspect_renderer.py`（仅如需展示季号范围）
- `tests/pipeline/test_baseline_tracking.py`
- `tests/reports/test_export_writer.py`（仅如报告展示变更）
- `tests/reports/test_inspect_renderer.py`（仅如报告展示变更）

## 禁止修改范围
- 🚫 AGENTS.md、CLAUDE.md
- 🚫 DB schema / migrations
- 🚫 `content_updates` 唯一索引
- 🚫 新集更新追踪逻辑

## 相关上下文
当前最小复现：

```json
{"events": 2, "written": 1, "rows": [["new_season:vs_1:s2", "{\"season\": 2}"]]}
```

S3 事件被 `insert or ignore` 和唯一索引静默丢弃。P1.9-hotfix-C 只修复了 `local_max_season` 回写到最大季号，没有修复 content_update 展示信息。

## 输入
- `NewSeasonEvent` 列表
- `content_updates` 当前 schema

## 输出
- 同一 virtual_series 的多个新季合并写入一条 content_update
- `source_summary_json` 至少包含：
  - `tmdb_tv_id`
  - `season`（向后兼容，建议取最大季号）
  - `seasons`（完整列表，如 `[2, 3]`）
  - `season_min`
  - `season_max`
  - `detected_at`

## 具体要求
1. `write_content_updates()` 写库前按 `virtual_series_id` 分组。
2. 每组写一条 `new_season` content_update，避免受唯一索引丢掉信息。
3. `content_update_id` 可使用最大季号或范围，但必须幂等稳定。
4. 旧报告读取 `source_summary_json["season"]` 仍能工作。
5. 如报告展示可低成本增强，显示 `S2-S3`；否则至少 JSON/inspect detail 可看到 `seasons`。
6. 新集更新追踪不进入本任务。

## 验收标准
1. ✅ local=1, tmdb=3 生成的两个事件写入 1 条 content_update，但 summary 包含 `[2, 3]`。
2. ✅ 单新季场景 summary 仍包含 `season: 2`。
3. ✅ 重复执行仍幂等，不重复写。
4. ✅ 全量测试通过。

## 风险点
- 不改唯一索引意味着“同一剧同一 update_type 只保留一条”仍是系统约束；本任务只确保这条记录信息完整。
- 如果未来要记录每一季独立 review 状态，需要另立 schema/产品任务。
