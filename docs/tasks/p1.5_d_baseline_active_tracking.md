## 任务名称

实现 ADR-0007 § 决策一的"功能 2"：每日按 TMDb `status` 智能轮询 A 库已有 TV 剧集，检测新季发布，写入 `content_updates`。电影跳过。

## 任务类型

feat（新增核心功能模块 + 调度逻辑 + CLI 集成）

## 当前阶段

Phase 1.5（V1 定位翻转，ADR-0007）

## 背景

- P1.5-C 完成后，`virtual_series` 表已有数据，每条记录对应一个 TMDb 剧集
- A 库（`upstream_programs`）提供了本地的节目清单（季级），`modify_instant` 可用于增量感知
- 功能 2 不受 `hot_score` 约束——只要 A 库有该剧，TMDb 显示新季就是有效更新

### 关键设计决策（已定）

| 问题 | 决策 | 理由 |
|------|------|------|
| **Q6：写入粒度** | 季级；TMDb `number_of_seasons` > `local_max_season` → 新季 | A 库数据模型为季级，写入粒度保持一致 |
| **Q7：增量检测** | `virtual_series.last_polled_at` + `upstream_programs.modify_instant` 双信号 | modify_instant 定位变更的节目，last_polled_at 控制轮询频率 |
| **Q8：集成方式** | **(c) 两者都做**：独立 CLI `baseline-track` + 嵌入 `daily-discover` 最后一步 | 排错可用独立命令，日常跑 daily-discover 自动包含 |

## 目标

每日自动检测 A 库 TV 剧集的新季发布：

1. `poll_scheduler` 根据 `poll_priority` 生成当日轮询计划
2. `baseline_tracking` 按计划调 TMDb，对比 `tmdb_number_of_seasons` vs `local_max_season`
3. 检测到新季 → 写入 `content_updates`，更新 `local_max_season`
4. 更新 `last_polled_at`

## 非目标

- ❌ 不追踪电影（功能 2 跳过 content_type='movie'）
- ❌ 不追踪集级别更新（episode-level 留 V2）
- ❌ 不修改 hot_score 计算
- ❌ 不输出到飞书（飞书已从链路移除）
- ❌ 不写日报（属 P1.5-F）

### 1. 智能轮询调度器（`src/movietrace/pipeline/poll_scheduler.py`）

```python
@dataclass
class PollPlan:
    virtual_series_id: int
    tmdb_tv_id: str
    name: str
    poll_priority: str        # urgent / normal / low / skip
    last_polled_at: str | None

def build_daily_poll_plan(conn, config: dict) -> list[PollPlan]:
    """按 poll_priority 分层 + last_polled_at 升序，生成今日轮询列表。

    算法：
    1. 按 poll_priority 分组（urgent / normal / low / skip）
    2. skip 直接跳过
    3. 各组按 last_polled_at ASC（NULL 最优先），取 quota 条
       - urgent: total / urgent_coverage_days (默认 14)
       - normal: total / normal_coverage_days (默认 21)
       - low: total / low_coverage_days (默认 180)
    4. 总量不可超过 daily_max_calls（默认 50）
    5. 返回 PollPlan 列表
    """
```

## 风险点

1. **首次运行可产生大量新季事件** — 如果 V1 运行后过去数月，多部剧可能已更新多季。对策：首次运行使用 `--dry-run` 先看量，再实际写入。
2. **TMDb API 配额** — `daily_max_calls=50`，远低于 50 次/秒上限，安全。对策：硬上限 + 超额推迟到次日。
3. **去重依赖 SELECT-before-INSERT** — 不在 schema 层加复合唯一索引（避免改 migration）。对策：写入前查重。
4. **给 P1.5-F 的交接** — `update_type='new_season'` 需在日报中展示。
