## 当前阶段

## 来源任务
来自 2026-05-14 OMDb API key 过期/配额耗尽问题。用户申请了新 key `c9c22b79`，要求代码支持多 key。

## 目标
支持 OMDb 多个 API key 轮转：当一个 key 返回 401/403（配额耗尽或过期）时，自动切换到下一个可用 key。

## 非目标
- ❌ 不改变 OMDb 请求逻辑（仍然按 candidate 逐个请求）。
- ❌ 不改变评分权重或 IMDb 评分计算。
- ❌ 不影响 TMDb/Trakt/FP 的 key 管理。
- ❌ 不实现跨日 key 状态记忆（每天重新尝试所有 key）。

## 具体要求

### 1. secrets 格式变更

```json
// 旧格式
"omdb": {"api_key": "xxx"}

// 新格式
"omdb": {"api_keys": ["c9c22b79", "old_key"]}
```

要求：
- 列表顺序即尝试顺序（新 key 排第一，优先使用）。
- 兼容旧格式：如果还是 `api_key` 字符串，自动转为单元素列表，不崩溃。

### 2. enrich_with_omdb() 签名变更

```python
def enrich_with_omdb(
    conn: sqlite3.Connection,
    candidates: list[MergedCandidate],
    omdb_api_keys: list[str],  # 原 str → list[str]
    ...
```

要求：
- 内部维护当前 key index。
- 当某个 key 收到 401 或 403 时，切换到下一个 key 重试**当前失败的 candidate**（不跳过）。
- 所有 key 都失败时，该 candidate 跳过，继续处理后续 candidate（但不再尝试已失效的所有 key）。
- 429（限流）不切换 key，退避重试。

### 3. key 失效追踪

`enrich_with_omdb()` 内部维护两个集合：
- `active_keys` — 尚未被标记失效的 key
- `dead_keys` — 已返回 401/403 的 key

第一次 401/403 后，该 key 从 `active_keys` 移除。后续 candidate 只用 `active_keys` 中的 key。

### 4. 日志和统计

返回 dict 增加字段：
```python
{
    "api_calls": int,
    "cache_hits": int,
    "enriched": int,
    "keys_used": int,          # 实际使用的 key 数量
    "keys_exhausted": int,     # 被标记失效的 key 数量
}
```

logger.info 输出使用的 key 数量和失效情况。

### 5. 调用方修改

`discovery.py::run_discovery()`：
```python
# 旧
omdb_key = (secrets.get("omdb") or {}).get("api_key", "")
if omdb_key:
    enrich_with_omdb(conn, candidates, omdb_key, ...)

# 新
omdb_keys = (secrets.get("omdb") or {}).get("api_keys") or []
if (secrets.get("omdb") or {}).get("api_key"):  # 兼容旧格式
    omdb_keys = [(secrets.get("omdb") or {}).get("api_key")]
if omdb_keys:
    enrich_with_omdb(conn, candidates, omdb_keys, ...)
```

`cli.py` 两处同理。

## 验收标准
1. ✅ 第一个 key 可用时，不使用第二个 key。
2. ✅ 第一个 key 401 时，自动切换到第二个 key 重试当前 candidate。
3. ✅ 所有 key 失效后，剩余 candidate 不再尝试。
4. ✅ 兼容旧 secrets 格式（`api_key` 字符串）。
5. ✅ 相关测试通过。

## 测试要求
- 单 key 可用 → 正常跑完。
- 第一个 key 401，第二个 key 200 → 切换到第二个，数据正确。
- 两个 key 都 401 → 返回 `keys_exhausted=2`，不崩溃。
- 旧格式 `api_key` 字符串兼容。
- 新格式 `api_keys` 列表正常工作。

PYTHONPATH=src .venv/bin/python -m pytest tests/test_omdb_enrichment.py -v

## 风险点
- 免费 key 每日配额有限（1000 次），仍可能在大量 candidate 时耗尽。
- 如果所有 key 同时失效，fallback 到 TMDb vote_average（P1.8-G 已有）。
- 旧格式兼容逻辑应在 secrets 迁移后清理（另排任务）。
