# 任务包：P1.40 fix-json-export-missing-fields

**任务包版本：** v1
**创建时间：** 2026-05-21 +08

## 任务名称
P1.40 修复 format_json 缺失字段——热点发现表中文名/播出平台/总集数为空

## 任务类型
fix

## 当前阶段
Phase 1（V1 运行观察期）

## 来源任务
用户反馈：今日 08:00 cron 跑完后飞书"热点发现"子表中 中文名/中文简介/播出平台/A库总集数/TMDB总集数 仍为空。

## 目标

修复 `export_writer.py:format_json()` 缺失 8 个字段的 bug，使 `sync.py` 能从 JSON 读取并写入飞书对应列。

## 根因

`format_json()` 的导出 dict 从未包含以下字段，而这些字段均已存在于 `_load_content_updates()` 的 SQL 查询结果中（分别在 P1.28 和 P1.33 加入 SQL，但未同步 `format_json`）：

| 缺失字段 | sync.py 使用目的 | 飞书列 |
|---|---|---|
| `content_type` | `_derive_content_type()` 首选路径 | "类型"（初始值）|
| `match_confidence_low` | "是否低置信度" | "是否低置信度" |
| `title_zh` | 中文名 | "中文名" |
| `overview_zh` | 中文简介 | "中文简介" |
| `genres_json` | 类型（genres）字符串 | "类型" |
| `networks_json` | 播出平台字符串 | "播出平台" |
| `upstream_total_eps` | A库总集数 | "A库总集数" |
| `tmdb_total_episodes` | TMDB总集数 | "TMDB总集数" |

## 非目标
- ❌ 不修改 SQL 查询逻辑
- ❌ 不修改 sync.py 字段映射
- ❌ 不修改飞书表结构
- ❌ 不处理字段值本身为空（数据问题由上游负责）

- `tests/reports/test_export_writer.py`（补回归测试）

- 🚫 `src/movietrace/feishu/sync.py`

- 🚫 所有其他文件

## 相关上下文
- `format_json()` 在 `src/movietrace/reports/export_writer.py:553`
- `_load_content_updates()` 的 SQL 在同文件 line 116，结果 dict 在 line 157
- `sync.py` 读 JSON 字段位置：`title_zh`（line 331）、`overview_zh`（line 332）、`genres_json`（line 333）、`networks_json`（line 334）、`upstream_total_eps`（line 335）、`tmdb_total_episodes`（line 336）
- `content_type` 由 `_derive_content_type(rec)` 读 `rec.get("content_type")`，有三级 fallback（update_type → content_update_id 解析），但直接提供更准确
- `match_confidence_low` 缺失导致"是否低置信度"永远显示"否"

## 输入
- 现有 `data/movietrace.db`（含 title_zh/overview_zh 等字段数据）

## 输出
- 修复后的 `src/movietrace/reports/export_writer.py`
- 更新后的 `tests/reports/test_export_writer.py`（含新回归测试）

在 `format_json()` 的 `export.append({...})` 末尾追加 8 个字段：

```python

"match_confidence_low": u.get("match_confidence_low"),
"title_zh":            u.get("title_zh"),
"overview_zh":         u.get("overview_zh"),
"genres_json":         u.get("genres_json"),
"networks_json":       u.get("networks_json"),
"upstream_total_eps":  u.get("upstream_total_eps"),
"tmdb_total_episodes": u.get("tmdb_total_episodes"),
```

不改动其他字段，不改动函数签名。
