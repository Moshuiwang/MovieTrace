# 任务包：P1.9-hotfix-D — API logging 脱敏加固

**任务包版本：** v1
**创建时间：** 2026-05-14 +08

## 任务名称
P1.9-hotfix-D：API usage 日志脱敏不完整加固

## 任务类型
fix

## 当前阶段
Phase 1.9：code review hotfix

## 来源任务
Code review CR-008（`reports/code_review_2026-05-14.md` § CR-008）

## 目标
加固 API usage log 的脱敏：修复 metadata key 归一化不匹配问题，对 `error_message` 也做脱敏。

## 非目标
- ❌ 不改变日志写入架构
- ❌ 不重新设计脱敏框架
- ❌ 不影响主链路性能

## 允许修改范围
- `src/movietrace/logging/api_usage.py` → `_sanitize_metadata()` + `log_api_call()`
- `src/movietrace/sources/http.py` → `_log_error()` 的 error_message 脱敏
- `tests/test_api_usage_logging.py`

## 禁止修改范围
- 🚫 AGENTS.md、CLAUDE.md
- 🚫 其他 source 文件

## 相关上下文
- 当前 `_sanitize_metadata` 对 key 归一化去除了 `_` 和 `-`，但禁用集合中保留了原始格式（如 `access_token`），导致 `accesstoken` ≠ `access_token` 匹配失败
- `error_message` 直接传 `str(exc)`，如果 URL 中含 apikey 参数会被记录

## 输入
- `src/movietrace/logging/api_usage.py`

## 输出
- 修复后的脱敏逻辑

## 具体要求
1. `_sanitize_metadata` 的 key 归一化改为：去除非字母数字后小写比较
2. 禁用集合中的 key 也以同样方式归一化存储
3. `log_api_call()` 中 `error_message` 写入前做 key/URL 脱敏扫描
4. `http.py::_log_error()` 传递 error_message 前过滤 URL 中的敏感参数

## 验收标准
1. ✅ `access_token` → `accesstoken` 能在禁用集合中被匹配到
2. ✅ `error_message` 包含 `apikey=xxx` 时 xxx 被脱敏
3. ✅ 测试覆盖新增脱敏场景
4. ✅ 全量测试通过

## 测试要求
- 更新 `tests/test_api_usage_logging.py`：
  - metadata 含 `access_token` → 被过滤
  - metadata 含 `client_secret` → 被过滤
  - error_message 含 `apikey=secret123` → `secret123` 被脱敏

## 验证命令
```bash
PYTHONPATH=src python -m pytest tests/test_api_usage_logging.py -v
PYTHONPATH=src python -m pytest tests/ -v
```

## 风险点
- 脱敏可能误伤正常字段名，但 metadata 字段名通常不含敏感词
- 正则脱敏性能影响可忽略（日志写入不频繁）
