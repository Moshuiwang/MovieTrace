## 任务名称
P1.55 修复 TMDb canonical-first 早退条件过宽，避免跳过必要 detail 补全

## 任务类型
fix

## 当前阶段
Phase 1（V1 运行观察期）

## 来源任务
本地未推 4 个提交 review 发现：P1.52 `canonical-first-enrichment` 在 `canonical_items.genres_json` 非空时直接跳过 TMDb detail API，但 canonical 快速路径只应用 `genres_json/title_zh/overview_zh/networks_json`，没有补齐 `release_date/original_language/last_air_date/seasons/last_episode_to_air` 等 detail 字段。

## 目标
修正 `enrich_with_tmdb_details()` 的 canonical fast-path：只有候选对象已具备原有 skip 条件要求的 TMDb detail 字段时，才允许因 canonical enrichment 命中而跳过 API；否则必须继续走现有 `api_cache → API` 路径补齐 detail 数据。

- ❌ 不改变 `canonical_items` schema
- ❌ 不新增 migration
- ❌ 不修改 `tmdb_detail_cache.py` 的缓存机制
- ❌ 不改 scoring/export/feishu 同步逻辑

## 相关上下文
当前风险代码：
- `src/movietrace/pipeline/omdb_enrichment.py` 中 `enrich_with_tmdb_details()` 循环开头：
  - `_load_canonical_enrichment(...)`
  - 若 `genres_json` 非空则 `_apply_canonical_enrichment(...)`、`canonical_hits += 1`、`enriched += 1`、`continue`

原有 skip 条件：
- `c.tmdb_data.release_date`
- `c.tmdb_data.original_language`
- TV 还需要 `c.tmdb_data.last_air_date`

风险：
- FP-only / Trakt-only 候选可能已有 canonical genres，但当前候选内存缺日期/语言/TV 最新播出字段。
- 直接 `continue` 会跳过 `get_tmdb_detail_with_cache()`，影响评分、展示日期和 `row_duration_hours`。

## 具体要求
1. 提取或复用一个判断函数，表达“候选已具备足够 TMDb detail 字段，可以跳过 detail API”。
2. canonical enrichment 命中时先应用中文/类型/平台字段，但只有候选已满足原有 skip 条件时才 `continue`。
3. 对缺少 `release_date` / `original_language` 的候选，即使 canonical 有 `genres_json`，也必须调用 `get_tmdb_detail_with_cache()`。
4. 对 TV 候选，缺少 `last_air_date` 时，即使 canonical 有 `genres_json`，也必须调用 detail 路径。
5. canonical 字段只是预填、随后仍走 detail 路径时，不提前增加 `enriched`；`enriched` 只在“canonical 完整 skip”或“detail data 成功应用”时增加一次。
6. 保留 `canonical_hits` 统计，但语义需要清楚：只统计“canonical 命中并跳过 API”的次数；若仅应用 canonical 字段但仍调用 API，不计入 `canonical_hits`。

## 验收标准
1. ✅ canonical 有 `genres_json` 且候选已有完整 detail 字段时，API 不调用，`canonical_hits=1`。
2. ✅ canonical 有 `genres_json` 但候选缺 `release_date` 时，API 会调用，`canonical_hits=0`。
3. ✅ canonical 有 `genres_json` 但候选缺 `original_language` 时，API 会调用，`canonical_hits=0`。
4. ✅ canonical 有 `genres_json` 但 TV 候选缺 `last_air_date` 时，API 会调用，`canonical_hits=0`。
5. ✅ canonical 字段预填后继续走 API 时，`enriched` 不重复计数。
6. ✅ 原有 TMDb detail enrichment 测试通过。
7. ✅ 全量测试通过。
8. ✅ `daily-discover --dry-run` 正常完成。
