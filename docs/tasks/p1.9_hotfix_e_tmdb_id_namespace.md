# 任务包：P1.9-hotfix-E — TMDb movie/tv ID 命名空间碰撞防护

**任务包版本：** v1
**创建时间：** 2026-05-14 +08

## 任务名称
P1.9-hotfix-E：TMDb external_ids 的 movie/tv 命名空间隔离

## 任务类型
fix

## 当前阶段
Phase 1.9：code review hotfix

## 来源任务
Code review CR-006（`reports/code_review_2026-05-14.md` § CR-006）

## 目标
TMDb 的 movie 和 TV 是不同命名空间，同一数字 ID 可同时代表一部电影和一部剧。当前 `external_ids` 只按 `(source, external_id)` 唯一，且查询不加 media_type，存在错链风险。

## 非目标
- ❌ 不修改 canonical_items 表结构
- ❌ 不重新设计 external_id 体系
- ❌ 不处理 Trakt/IMDb 等其他 source 的命名空间

## 允许修改范围
- `src/movietrace/db/migrations/011_external_id_granularity.sql`
- `src/movietrace/db/schema.py`
- `src/movietrace/pipeline/discovery.py` → `_lookup_canonical_id` + `_ensure_canonical_item`
- `src/movietrace/pipeline/entity_matching.py` → 相关 external_ids 写入处
- `tests/db/test_schema_migration_011.py`

## 禁止修改范围
- 🚫 AGENTS.md、CLAUDE.md
- 🚫 canonical_items 表结构
- 🚫 评分链路

## 相关上下文
- TMDb movie ID 和 TV ID 是独立自增的，存在碰撞可能（虽然概率低但存在）
- `external_ids` 唯一键：`ux_external_ids_source_id(source, external_id)`
- `_lookup_canonical_id` 只查 `where source='tmdb' and external_id=?`
- 修复方案：TMDb 的 external_id 改为 `tv:{id}` / `movie:{id}` 前缀格式

## 输入
- `data/movietrace.db`
- 现有 external_ids 表中的 tmdb 记录

## 输出
- migration 011：更新现有 TMDb external_id 加前缀
- 修复后的 lookup / insert 逻辑

## 具体要求
1. migration 011：将现有 `source='tmdb'` 的 `external_id` 更新为带前缀格式
   - 已有 `external_granularity` 字段可用作判断依据
   - 无 granularity 的记录需通过 canonical_items.content_type 推导
2. `_lookup_canonical_id(conn, tmdb_id, media_type)` 增加 media_type 参数
3. `_ensure_canonical_item` 写入时使用前缀格式
4. entity_matching.py 中写入 external_ids 处同步更新

## 验收标准
1. ✅ TMDb external_id 统一为 `tv:{id}` / `movie:{id}` 格式
2. ✅ `_lookup_canonical_id` 区分 movie/tv
3. ✅ 既有数据 migration 后格式正确
4. ✅ 全量测试通过

## 测试要求
- 新增 migration 011 测试
- 更新 discovery 相关测试

## 风险点
- migration 需要推导无 granularity 记录的 media_type
- 调用 `_lookup_canonical_id` 的地方需要传入 media_type（当前约 2-3 处）
- 若既有数据量大，migration 可能耗时（当前 ~450 条 tmdb external_ids，秒级完成）
