## 任务名称

新增 `tmdb_trending` 和 `trakt_trending` 表,作为 TMDb / Trakt 热度榜单的独立存储,与 `flixpatrol_top10` 并列、互不干扰。

## 任务类型

feat (schema migration)

## 当前阶段

Phase 1.7(多热门源扩充)

## 来源任务

Phase 1.7 总体规划 — 用户诉求 #2「三个热门源独立保存,互不干扰」

## 目标

通过 migration 007 新增两张表,为 P1.7-B/C 准备存储结构:

- `tmdb_trending` — 存 TMDb 热度榜单条目(trending/all/day + tv/popular + movie/popular)
- `trakt_trending` — 存 Trakt trending 条目(shows + movies)

## 非目标

- ❌ 不修改 `flixpatrol_top10`
- ❌ 不实现数据采集逻辑(P1.7-B/C)
- ❌ 不改 `content_updates` 或 `candidates` 表

## 允许修改范围

- `src/movietrace/db/schema.py` — 追加 migration 007
- `tests/db/test_schema_migration_007.py` — 新建

## 禁止修改范围

- 🚫 现有表结构(`flixpatrol_top10` / `candidates` / `content_updates` / `canonical_items` 等)
- 🚫 已执行的 migration 001-006

## 相关上下文

- 调研结论:
  - TMDb 候选范围:trending/day + tv/popular + movie/popular 各 3 页 ≈ 180 条/天
  - Trakt 候选范围:shows/trending(≤500) + movies/trending(≤104) ≈ 730 条/天
- FP 数据有 `tmdb_id` 97% 填充率,合并以 tmdb_id 为主键

### tmdb_trending 表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK AUTOINCREMENT | |
| tmdb_id | INTEGER | NOT NULL | TMDb 唯一 ID |
| media_type | TEXT | NOT NULL | 'movie' / 'tv' |
| title | TEXT | NOT NULL | |
| original_title | TEXT | | |
| release_date | TEXT | | YYYY-MM-DD,TV 用 first_air_date |
| original_language | TEXT | | 'en' / 'es' / ... |
| popularity | REAL | NOT NULL | TMDb popularity 值 |
| vote_average | REAL | | |
| vote_count | INTEGER | | |
| source_endpoint | TEXT | NOT NULL | 'trending/day' / 'tv/popular' / 'movie/popular' |
| source_page | INTEGER | NOT NULL | 抓取的页码 |

| raw_payload_json | TEXT | NOT NULL | 原始 JSON |

索引:
- UNIQUE(`tmdb_id`, `media_type`, `source_endpoint`, `snapshot_date`) — 同一条目同一端点当天只存一次

- INDEX(`tmdb_id`)
