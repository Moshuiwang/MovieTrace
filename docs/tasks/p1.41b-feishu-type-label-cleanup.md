## 相关上下文
P1.41 新增“类型标签”多选字段后，当前 main 中 `_compute_type_labels()` 返回 `content_type + genre`，同时 `fields_extra["类型"] = _names_from_json(genres_json)` 会覆盖前面 `_derive_content_type(rec)` 写入的 `movie/tv`。

## 输入
- 当前 `src/movietrace/feishu/sync.py`
- 当前 `tests/test_feishu_sync.py`
- 关闭未合并的 PR #38 / 本地遗留分支 `fix/p1.41-type-label-cleanup`

## 输出
- 修正后的飞书字段映射
- 更新后的单元测试
- `STATE.md` 状态同步

## 具体要求
1. 删除 `fields_extra` 中对 `"类型"` 的 genre 覆盖。
2. `_compute_type_labels()` 只返回 TMDb genre name，不包含 `content_type`。
3. 无 `genres_json` 时返回空列表，不写“类型标签”。
4. 保持“类型”字段由 `_derive_content_type(rec)` 决定。

## 验收标准
1. ✅ movie + Action 时，“类型标签”返回 `["Action"]`。
2. ✅ tv + Drama/Science Fiction 时，“类型标签”返回 `["Drama", "Science Fiction"]`。
3. ✅ 无 genre 时，“类型标签”返回 `[]`。
4. ✅ `sync_table()` 创建记录时，“类型”字段仍是 `movie/tv`，不被 genre 覆盖。
5. ✅ `tests/test_feishu_sync.py` 通过。
6. ✅ 全量测试通过。

## 测试要求
- 更新 `TestComputeTypeLabels` 4 个测试。
- 如现有 sync mapping 测试已覆盖“类型”，确保断言仍指向 `movie/tv`。
