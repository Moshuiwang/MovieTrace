# 任务包：P1.12-D — PyYAML 依赖声明补齐

**任务包版本：** v1
**创建时间：** 2026-05-14 15:05 +08

## 任务名称
P1.12-D：将 PyYAML 纳入 requirements，保证 config.yaml 可复现解析

## 任务类型
fix

## 当前阶段
Phase 1.12：review hotfix

## 来源任务
2026-05-14 Codex 全面代码 review：`cli.py` 和 `scoring.py` 依赖 `yaml`，但 `requirements.txt` 未声明 PyYAML；当前 `.venv` 只是碰巧存在 `PyYAML 6.0.3`。

## 目标
补齐 PyYAML 运行依赖，使全新环境执行 `pip install -r requirements.txt` 后能稳定读取 `config.yaml` 和 `config/scoring_weights.yaml`。

## 非目标
- ❌ 不替换配置格式。
- ❌ 不改 `config.yaml` 的业务值。
- ❌ 不改评分权重。
- ❌ 不做新集更新追踪。

## 允许修改范围
- `requirements.txt`
- `tests/test_scoring.py`
- `tests/test_discovery.py` 或新增配置加载测试文件（如有必要）
- `docs/tasks/p1.12_hotfix_d_pyyaml_dependency.md`

## 禁止修改范围
- 🚫 AGENTS.md、CLAUDE.md
- 🚫 `config.yaml`
- 🚫 `config/scoring_weights.yaml`
- 🚫 DB schema / migrations

## 相关上下文
- `src/movietrace/cli.py::_load_config()` 捕获所有异常并返回 `{}`；缺 PyYAML 时配置会静默失效。
- `src/movietrace/pipeline/scoring.py::load_weights_config()` 缺 PyYAML 时回退默认权重。
- 项目约束：不主动引入新依赖；本任务包即为引入/固定 PyYAML 的授权边界。

## 输入
- 当前 `.venv` 中实测 PyYAML 版本：`6.0.3`
- `requirements.txt`

## 输出
- `requirements.txt` 显式包含 `PyYAML==6.0.3`
- 配置加载测试或现有测试覆盖仍通过

## 具体要求
1. 在 `requirements.txt` 增加 `PyYAML==6.0.3`。
2. 如已有测试足够覆盖，可不新增测试；但完成报告必须说明覆盖依据。
3. 若新增测试，验证 `_load_config()` 或 `load_weights_config()` 能读取 YAML 中的关键字段。
4. 不修改真实配置值。

## 验收标准
1. ✅ `requirements.txt` 包含固定版本 PyYAML。
2. ✅ `.venv/bin/python -c "import yaml; print(yaml.__version__)"` 可输出版本。
3. ✅ 全量测试通过。

## 测试要求
- 至少运行评分和 discovery 相关测试。
- 如新增测试，避免依赖真实 secrets 或外部 API。

## 验证命令
```bash
.venv/bin/python -c "import yaml; print(yaml.__version__)"
PYTHONPATH=src .venv/bin/python -m pytest tests/test_scoring.py tests/test_discovery.py -v
PYTHONPATH=src .venv/bin/python -m pytest tests/ -v
```

## 风险点
- 如果部署环境已有不同 PyYAML 版本，固定版本可能触发重装；需在完成报告中注明。
- `_load_config()` 当前静默吞异常的问题不在本任务内扩大处理。
