## 具体要求
1. `v1_closeout_review.md` 至少包含：
   - V1 最终目标。
   - 实际完成能力。
   - 与最初设想相比的关键方向变化。
   - 当前系统能稳定做什么。
   - 明确不能做什么。
   - 已知阻塞和剩余风险。
   - 为什么暂不启动 V2。
2. `runbook.md` 至少包含：
   - 首次接手检查命令。
   - 数据库备份和 schema 检查命令。
   - 每日运行命令。
   - dry-run 与 commit 模式区别。
   - secrets 路径。
   - 常见失败：FP 402、OMDb key、TMDb/Trakt 请求失败、fallback。
   - 如何导出给运营看的结果。
3. `feedback_log_template.md` 至少包含：
   - 日期。
   - 运行命令和数据源状态。
   - 候选数量、采纳数量、误报、漏报。
   - 运营备注。
   - 是否触发 V2 候选需求。
4. 修正 `SCOPE.md` 中与当前实现不一致的飞书描述：
   - 不得把飞书写入描述为当前实现。
   - 如保留“中间表”概念，应说明当前实现是本地 MD/JSON 导出，未来协作界面替换属 V2 或后续决策。
5. `STATE.md` 中记录 P1.15 文档产物和当前 V1 运行观察期建议。
6. 文档应简洁，避免复制大段历史；历史细节用链接指向已有 ADR/STATE。
