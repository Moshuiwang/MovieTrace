## 具体要求

### 1. sync_table 入口 ensure
在 `sync_table` 拿到 token 后、写数据前，加：
```python
ensure_result = ensure_table_fields(
    app_id=app_id, app_secret=app_secret,
    app_token=app_token, table_id=table_id,
)
if ensure_result["created"]:
    _notify_fields_created(ensure_result, ...)
if ensure_result["errors"]:
    _notify_fields_error(ensure_result, ...)
    raise RuntimeError(...)   # 终止 sync
```

### 2. 通知 helper（notify.py 新增）
- `_notify_fields_created(stats, receive_id, webhook, ...)`: 发"已补建 N 个字段：xxx"卡片或文本
- `_notify_fields_error(stats, ...)`: 发报警，含 error 列表

### 3. TEMPLATE.md
"完成后输出要求"段末尾追加：
```
完成后按 CLAUDE.md § Git 工作流 交付（feature branch → PR）。
```

### 4. pre-push hook
`scripts/hooks/pre-push`（可执行）：
- 读 stdin（每行 `local_ref local_sha remote_ref remote_sha`）
- 任一行 `remote_ref` 为 `refs/heads/main` → 打印多行清晰提示 → `exit 1`
- 否则 `exit 0`

本次执行时跑一次 `git config core.hooksPath scripts/hooks` 配置 local。

### 5. 测试
- mock `ensure_table_fields` 返回 `created=[xxx]`/`errors=[xxx]`/`created=[]` 三种情况
- 验证：created 非空 → 调用通知；errors 非空 → 抛错；ensure 调用顺序在写数据之前
- pre-push hook 用 bash 测试：`echo "ref refs/heads/main" | bash scripts/hooks/pre-push` 应 exit 1
