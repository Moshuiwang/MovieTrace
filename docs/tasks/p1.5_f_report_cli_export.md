## 任务名称

合并原 P1.5-F（日报模板 + CLI 语义）和 P1.5-G（检测与导出解耦），并适配"飞书移除"后的新架构：
- 日报模板从 A 库数据驱动
- CLI 语义与"更新追踪系统"定位对齐
- 新增 `export-recommendations` 命令（输出 MD+JSON 文件取代飞书）

## 任务类型

feat + refactor

## 当前阶段

Phase 1.5（V1 定位翻转，ADR-0007）

## 背景

- **飞书已从系统链路移除**。原来的"写入飞书建议更新表"改为"导出为本地报告文件"
- 数据源头现在是 A 库（`upstream_programs` / `upstream_episodes`）+ `content_updates` 表
- ADR-0007 § 七 要求检测与导出解耦：`daily-discover` 只做检测，`export-recommendations` 按需导出
- 日报仍然需要，但数据源和语义变了

## 目标

1. **日报模板更新** — 基于 `content_updates` + `upstream_programs` 生成日报，不再依赖飞书
2. **CLI 语义对齐** — 命令名和输出措辞与"更新追踪系统"定位一致
3. **导出命令** — `export-recommendations` 从 B 库读取 content_updates，输出为 MD 报告 + JSON 数据文件到 `reports/` 目录
4. **飞书代码清理** — 移除飞书写入相关的死代码

## 非目标

- ❌ 不新增飞书相关功能
- ❌ 不修改 `hot_score` 算法
- ❌ 不改 entity_matching / baseline_tracking 逻辑
- ❌ 不引入新依赖
