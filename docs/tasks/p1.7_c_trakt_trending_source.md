## 相关上下文

### Cloudflare 1010 问题(实测发现)

Trakt API 现在对默认 Python UA 返回 403 Cloudflare 1010 (browser signature blocked)。**必须使用浏览器 UA**:

```python
'User-Agent': 'Mozilla/5.0 (X11; Linux x86_64; rv:109.0) Gecko/20100101 Firefox/115.0'
```

**当前 `src/movietrace/sources/http.py` 不设默认 UA,导致 Trakt 调用全部失败**。本任务需修复 `get_json` 的默认 UA(对所有源生效,不止 Trakt)。

### Trakt API key

- `client_id` 路径:`/tmp/movietrace_phase0_secrets.json` → `trakt.client_id`
- 需要 header:`trakt-api-key: <client_id>`, `trakt-api-version: 2`

### 字段映射

trending 响应结构:
```json
[
  {
    "watchers": 7810,
    "show": {
      "ids": {"trakt": 139960, "tmdb": 76479, "imdb": "tt1190634"},
      "title": "FROM",
      "year": 2022,
      "rating": 7.99,
      "votes": 10182
    }
  }
]
```

movies/trending 用 `"movie"` 而非 `"show"`。

## 具体要求

### http.py 改造

修改 `get_json` 函数,添加默认 UA:

```python
DEFAULT_USER_AGENT = "Mozilla/5.0 (X11; Linux x86_64; rv:109.0) Gecko/20100101 Firefox/115.0"

def get_json(url, *, params=None, headers=None, timeout=20):
    merged_headers = {"User-Agent": DEFAULT_USER_AGENT}
    if headers:
        merged_headers.update(headers)
    ...
```

调用方传的 headers 优先级高于默认值(允许覆盖)。

### TraktTrendingClient 接口

```python
class TraktTrendingClient:
    def __init__(self, client_id: str, base_url: str = "https://api.trakt.tv"):
        ...

    def fetch_shows_trending(self, limit: int = 500) -> list[dict]:
        """GET /shows/trending?limit=N&extended=full"""

    def fetch_movies_trending(self, limit: int = 500) -> list[dict]:
        """GET /movies/trending?limit=N&extended=full"""
```

返回的每个 dict 应当**已经扁平化**为:
```python
{
    "watchers": int,
    "trakt_id": int,
    "tmdb_id": int | None,
    "imdb_id": str | None,    # 保留 tt 前缀
    "title": str,
    "year": int | None,
    "rating": float | None,
    "votes": int | None,
    "media_type": "show" | "movie",
    "raw_payload": dict,
}
```

### 采集编排

```python
def fetch_and_store_trakt_trending(
    db_path: str,
    client_id: str,
    snapshot_date: str | None = None,
) -> dict:
    """
    1. 调 shows/trending limit=500
    2. 调 movies/trending limit=500
    3. INSERT OR IGNORE(UNIQUE 约束去重)
    4. 返回 {fetched, inserted, errors}
    """
```

### 礼貌采集

- 两次调用之间 sleep 1 秒

### 错误处理

- 单端点失败:记录日志,继续另一端点
- 403 错误必须明确报错(说明 UA 问题),不能静默
- 缺 tmdb_id 的条目仍入库(用 trakt_id 作主键),不丢弃

## 风险点

- **修改 `get_json` 默认 UA 可能影响 FP / OMDb / TMDb**:这三个源目前用默认 UA(空)能通,加上 Mozilla UA 理论上更"合规"。**风险点**:FP 任务包(`reference/phase0_supplement.md`)规定 UA = `MovieTraceBot/0.1`,FP 调用方应该已经显式传了自己的 UA(检查 `flixpatrol_api.py`)。本任务的默认 UA 仅作为兜底,**不影响已显式传 UA 的调用方**
- **Trakt 速率限制**:Trakt 文档说每秒 ~1 次,本任务 2 次/天完全无压力
- **`limit=500` 会不会被砍到上限**:实测 shows 返回 500 条(实际 trending 总量 626),movies 返回 104 条(全量),无问题
