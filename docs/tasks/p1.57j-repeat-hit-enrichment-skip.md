**任务包版本：** v1
**创建时间：** 2026-05-24 00:53 +08

## 任务名称
P1.57j 旧 discovery 重复命中默认跳过重 enrichment

## 任务类型
refactor

## 当前阶段
Phase 1（V1 运行观察期）

## 来源任务
ADR-0016 规定已存在的内容再次命中时默认跳过重 enrichment，只写 observation 并更新当前项最近状态字段。

## 目标
当 discovery 稳定键已存在且 metadata 足够时，`daily-discover` 对重复命中跳过昂贵 enrichment，只写 observation 和 current 最新状态。

## 非目标
- ❌ 不改首次发现 enrichment
- ❌ 不改 TMDb / OMDb 客户端
- ❌ 不改变评分公式
- ❌ 不新增缓存表
- ❌ 不跳过 observation 写入

## 允许修改范围

- `src/movietrace/pipeline/omdb_enrichment.py`
- `tests/test_discovery*.py`
- `tests/test_omdb_enrichment.py`
- `STATE.md`

## 禁止修改范围

- 🚫 `src/movietrace/feishu/**`
- 🚫 `src/movietrace/reports/**`

## 相关上下文
- P1.55 已修复 canonical-first early skip：canonical_items 不能因有 genres 就单独触发 TMDb detail API skip。
- 本任务只能跳过重复命中内容的重 enrichment，不能让首次发现缺字段。

## 输入
- current discovery item 存在性
- 候选当前字段完整度
- P1.55 TMDb detail skip 回归场景

## 输出
- 重复命中 enrichment skip 判断
- stats 中 skip 计数
- 回归测试

## 具体要求
1. 首次发现仍走完整 enrichment。
2. 重复命中如果已有稳定 metadata 且候选不缺必要输出字段，默认跳过 OMDb / TMDb detail 重 enrichment。
3. 重复命中如果缺必要字段，仍允许补 detail，不能因 current item 存在而错误早退。
4. `canonical_hits` 口径保持清楚：只统计 canonical 命中且真的跳过 TMDb detail API 的次数。
5. stats 输出重复命中跳过数量，例如 `repeat_hit_enrichment_skipped`。

## 验收标准
1. ✅ 旧 discovery 再次命中时 observation 写入，重 enrichment 可被跳过。
2. ✅ 首次发现仍调用必要 enrichment。
3. ✅ 缺 release_date / original_language / TV last_air_date 等必要字段时仍调用 TMDb detail。
4. ✅ P1.55 回归测试保持通过。

## 测试要求
- 覆盖首次发现、重复命中 metadata 完整、重复命中 metadata 缺字段。
- 覆盖 `canonical_hits` 计数不被误增。
