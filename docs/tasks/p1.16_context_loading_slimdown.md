# 任务包：P1.16 — 上下文加载规则与文档瘦身

**任务包版本：** v1
**创建时间：** 2026-05-14 20:18 +08

## 任务名称
P1.16：上下文加载规则与文档瘦身

## 任务类型
docs

## 当前阶段
V1 维护/收尾

## 来源任务
2026-05-14 工作方法复盘：当前文档体系保障了工程连续性，但启动时读取大量历史文档造成 token 成本过高，需要改为分层、按需加载。

## 目标
建立轻量上下文入口和分层加载规则，压缩 `STATE.md` 的启动负担，并将历史状态迁移到归档文档中。

## 非目标
- ❌ 不删除历史资料。
- ❌ 不改业务代码。
- ❌ 不改变 V1/V2 产品边界。
- ❌ 不重写所有 ADR、requirements 或 product roadmap。
- ❌ 不在本任务中归档所有旧任务包；如需要，另开任务包。

## 允许修改范围
- `docs/context_map.md`（新建）
- `docs/history/phase1_state_archive.md`（新建）
- `STATE.md`
- `AGENTS.md`
- `CLAUDE.md`
- `journal/`（会话收尾日报，如本任务执行方需要）

## 禁止修改范围
- 🚫 `src/`
- 🚫 `tests/`
- 🚫 `data/`
- 🚫 `docs/decisions/`（除非只修链接）
- 🚫 `docs/tasks/` 中既有任务包正文
- 🚫 `requirements.txt`

## 相关上下文
- 当前启动顺序要求读取 `STATE.md`、AGENTS 规则、最近日报、任务包。
- `STATE.md` 已包含大量 Phase 1 历史，作为项目年鉴有价值，但不适合每次完整加载。
- AGENTS.md 与 CLAUDE.md 内容等价，修改入口规则时两边必须同步。

## 输入
- `STATE.md`
- `AGENTS.md`
- `CLAUDE.md`
- `SCOPE.md`
- `docs/tasks/TEMPLATE.md`
- `docs/workflow/`

## 输出
- `docs/context_map.md`：当前 Agent 的按需加载地图。
- `docs/history/phase1_state_archive.md`：Phase 1 历史状态归档。
- 精简后的 `STATE.md`：保留当前状态、阻塞项、下一步、关键链接，历史细节改为指向归档。
- 同步更新后的 AGENTS.md / CLAUDE.md 启动顺序和 token 预算规则。

## 具体要求
1. 新增 `docs/context_map.md`，定义四层文档：
   - 当前层：默认读取。
   - 操作层：运行、排障、反馈时读取。
   - 决策层：边界争议或产品判断时读取。
   - 历史层：默认不整篇读取，先 `rg` 搜索再打开命中片段。
2. `STATE.md` 精简目标：
   - 保留当前阶段表。
   - 保留进行中/待执行任务。
   - 保留阻塞项、待用户决策、V2 backlog。
   - 保留真实库 schema 状态和最新验证结果。
   - 大段 Phase 1 历史迁移到 `docs/history/phase1_state_archive.md`。
3. `STATE.md` 不要求强行压到固定行数，但应明显降低启动读取成本；目标 150-250 行左右。
4. AGENTS.md / CLAUDE.md 同步增加：
   - 启动默认读取 `STATE.md`、`SCOPE.md`、`docs/context_map.md`、当前任务包。
   - 不默认整篇读取历史日报、已完成任务包、完整 requirements。
   - 查历史先 `rg` 搜索关键词，再打开小段。
5. 不得丢失历史事实；迁移内容应能从 `STATE.md` 链接到归档文档。
6. 修改 AGENTS.md / CLAUDE.md 时必须保持两者入口规则内容等价。

## 验收标准
1. ✅ `docs/context_map.md` 存在，并能指导 Agent 按需加载。
2. ✅ `docs/history/phase1_state_archive.md` 存在，保留迁移出的 Phase 1 历史。
3. ✅ `STATE.md` 明显精简，且仍能回答当前阶段、下一步、阻塞项、真实库状态。
4. ✅ AGENTS.md 与 CLAUDE.md 的启动顺序和 token 预算规则一致。
5. ✅ 关键历史关键词仍可通过 `rg` 找到。

## 测试要求
纯文档任务，不新增测试。

## 验证命令
```bash
rg -n "context_map|默认读取|按需|历史层|Phase 1|schema|阻塞项|V2 backlog" STATE.md AGENTS.md CLAUDE.md docs/context_map.md docs/history/phase1_state_archive.md
diff -u <(rg -n "context_map|默认读取|按需|历史" AGENTS.md) <(rg -n "context_map|默认读取|按需|历史" CLAUDE.md)
```

## 风险点
- 压缩 `STATE.md` 可能误删后续排障需要的上下文；必须迁移到归档而不是删除。
- 修改 AGENTS.md / CLAUDE.md 容易不同步；必须执行文本检查。
- 本任务会改变 Agent 启动习惯，完成后应在 `STATE.md` 明确提示。

## 完成后输出要求
按 [docs/workflow/report-format.md](../workflow/report-format.md) 格式汇报。
