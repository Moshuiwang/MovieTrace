### TMDb client 现状（src/movietrace/sources/tmdb.py）
- `get_movie_detail` / `get_tv_detail` 硬编码 `language=en-US`（行 159/174/226/274/289/305）
- `api_cache` 缓存 key 格式：`tmdb:detail:tv:{id}` / `tmdb:detail:movie:{id}`
- zh-CN 新 cache key：`tmdb:detail:tv:{id}:zh-CN` / `tmdb:detail:movie:{id}:zh-CN`

### 每日 API 配额影响
- 当前用量：先 `select count(*) from api_usage_log where service='tmdb' and request_date=date('now')` 确认基线
- 双语抓取后：每 candidate +1 次 TMDb detail（zh-CN），一次性回填 ~905 canonical_items
- TMDb 免费层每日 10,000 次，历史回填分批跑（建议每次 --limit 100，多天完成）

### 数据流
```
TMDb detail zh-CN → candidate.tmdb_data_zh（临时）
    ↓
enrich 结束 → 写入 canonical_items.title_zh / overview_zh
    ↓
genres / networks 从 en-US detail 提取 → canonical_items.genres_json / networks_json
    ↓
sync.py 从 canonical_items 读 4 列 → 飞书
```

## 具体要求

### A. Migration 017
```sql
-- src/movietrace/db/migrations/017_canonical_zh_fields.sql
ALTER TABLE canonical_items ADD COLUMN title_zh TEXT;
ALTER TABLE canonical_items ADD COLUMN overview_zh TEXT;
ALTER TABLE canonical_items ADD COLUMN genres_json TEXT;    -- JSON array: [{"id":18,"name":"Drama"},...]
ALTER TABLE canonical_items ADD COLUMN networks_json TEXT;  -- JSON array: [{"id":213,"name":"Netflix"},...]
```

### B. TMDb client
- `get_movie_detail(tmdb_id, language="en-US")` — 增加可选 language 参数，**默认不变**
- `get_tv_detail(tmdb_id, language="en-US")` — 同上
- cache_key：`f"tmdb:detail:{content_type}:{tmdb_id}:{language}"` — 原有英文缓存 key 变为 `...:en-US`，**不破坏**现有 24h 缓存（已存的 key 格式 `tmdb:detail:tv:{id}` 与新格式不同，视为 cache miss → 自然重建）

### C. enrich pipeline
- 在 `enrich_with_tmdb_details` 中：
  - 抓 zh-CN detail，取 `name`/`title` + `overview`，若非空写入 candidate
  - 从 en-US detail 取 `genres`（list of dicts）→ `json.dumps(genres, ensure_ascii=False)`
  - TV only：从 en-US detail 取 `networks` → `json.dumps(networks, ensure_ascii=False)`
  - 电影无 networks，写 NULL

### D. canonical_items 写入
- 在 canonical_items upsert 时追加 4 列（ON CONFLICT 用 `DO UPDATE SET ... title_zh=excluded.title_zh ...`）
- zh-CN 字段为空时写 NULL（不写空字符串）

### E. 飞书字段
- `schema_setup.py` 新增：中文名（文本）/ 中文简介（文本，多行）/ 类型（文本）/ 播出平台（文本）
- `sync.py` 从 canonical_items 4 列读取并写入飞书

### F. 回填脚本（scripts/p1_28_backfill_zh_fields.py）
- 遍历 canonical_items（支持 `--limit N` 分批）
- 对每条调 TMDb zh-CN detail（受 api_cache 保护）
- 写入 4 列
- 输出统计：updated / skipped（cache miss 或 zh title 为空）/ total

## 风险点

- **最大改动**：涉及 schema migration + 3 个模块 + 飞书字段。建议作为**独立 PR**，不与 p1.25/p1.26/p1.27 混合。
- **api_cache 缓存 key 变化**：现有 `tmdb:detail:tv:{id}` key 与新格式 `tmdb:detail:tv:{id}:en-US` 不匹配，已缓存的 en-US 数据会被重新抓取（cache miss 一次），消耗配额。评估：~905 条 canonical_items，一次性额外消耗 ~905 次，在 10k/日配额内。
- **TMDb zh-CN 覆盖率**：部分冷门内容可能无中文元数据，title_zh 为空 → 飞书字段空白。合理预期，不视为 bug。
- **回填分批**：--limit 100 每次，约需 10 次运行（或一次跑完若配额充足）。
