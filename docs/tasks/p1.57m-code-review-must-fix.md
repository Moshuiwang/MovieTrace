## 具体要求

### A1 — backfill 事务边界
1. `scripts/p1_57_backfill_current_discovery.py` 改为"按 item 提交"或"全量单事务、首个失败即 abort 整批"二选一，**必须显式说明选择哪种**并在代码注释里写明理由。
2. 不允许保留当前"单事务 + 中途 rollback + continue + 末尾 commit"的组合（会静默丢失中途 item 之前的写入）。
3. 选"按 item 提交"时：每个 item 处理完成后 `conn.commit()`；遇错 `conn.rollback()` 只回滚当前 item 的本地修改；`stats["errors"] += 1`；继续下一个 item。
4. 选"全量单事务 + abort"时：单个 item 失败立即 raise，整批回滚，main() 退出非 0；停止后人工 / 重跑。
5. `scripts/verify_p1_57_upgrade_rebuild.sh` 在 backfill `--commit` 之后必须**断言** `current_discovery_items` 和 `discovery_observations` 计数都 ≥ 预期值（预期值至少为去重后的 `new_discovery` 行的 discovery key 数量）；不符合即 `exit 1`。

### A2 — batch write 的 canonical_id 保护
1. `_write_current_discovery_batch` 在 `_lookup_canonical_id` 返回 `None` 时显式 `continue`，并把跳过条数计入新 stat `stats["skipped_no_canonical_id"]`。
2. 不允许写入 `canonical_item_id=NULL` 的 current_discovery_items 行（即使 schema 允许）。
3. `run_discovery` 把 `cd_stats["skipped_no_canonical_id"]` 透传到 `stats["current_discovery_skipped_no_canonical_id"]`。

### A3 — export `created_at` 语义
1. `_load_current_discovery_rows` 返回的字典中，移除 `"created_at": row["updated_at"]` 这一映射；不要把 `updated_at` 伪装成 `created_at`。
2. 若 markdown / json render 流程需要"事件时间"列，统一改用 `first_discovered_date`（首次）或 `last_discovered_date`（最近），并在 dict 中用这两个真实键。
3. 不允许任何 export 输出仍然把"今天再次被发现"误导为"今天首次发现"。
4. 现有 `format_markdown` 的"其他更新"表如继续渲染 `created_at`，必须改读 `last_discovered_date`（discovery 行）或保留原来的 `created_at`（new_season 行）；两类行的渲染必须可区分。

### A4 — 飞书 discovery 更新不覆盖首次语义
1. `src/movietrace/feishu/sync.py` discovery 更新分支（`existing_rec and ut == "new_discovery"`）构造 `system_fields` 时，必须从 base `fields` 里**剔除** `"发现日期"`（以及任何会覆盖"首次"语义的字段）。
2. 仅在 create 分支写入 `首次发现日期`；update 分支不动 `首次发现日期`、不动 `发现日期`。
3. update 分支允许写 `最近发现日期` 和 `发现次数`（保留 P1.57h 行为）。

## 验收标准
1. ✅ 模拟 backfill 在第 N 个 item raise，且 N 前已有成功写入：DB 中 N 前的 item 计数与单 item 提交模式一致，或脚本以非 0 退出且 DB 完全无变化（按所选策略二选一）。
2. ✅ `_write_current_discovery_batch` 单元测试覆盖 `canonical_id=None` 路径：不写 current_discovery_items，stat `skipped_no_canonical_id == 1`。
3. ✅ `_load_current_discovery_rows` 返回的 dict 中没有 `created_at` 键，或该键值不来自 `updated_at`。
4. ✅ `format_markdown` 对一条 discovery 行渲染的时间列，反映 `last_discovered_date`，不是当天最近一次 sync 时间。
5. ✅ 飞书 sync 单测：discovery update 路径调用 `batch_update_records` 的 fields 中不含 `首次发现日期`、不含 `发现日期`；含 `最近发现日期` 和 `发现次数`。
6. ✅ `verify_p1_57_upgrade_rebuild.sh` 跑通且 backfill 计数断言能在故意注入失败时 `exit 1`。
7. ✅ `PYTHONPATH=src python -m pytest tests/ -v` 全量通过；新增 / 修改的测试需对应到上述 A1-A4 每一条至少一个测试。

## 风险点
- A1 选"按 item 提交"会让一次 backfill 产生大量小事务，对 SQLite 写盘略慢；可接受（backfill 是一次性操作）。
- A1 选"全量 abort"则要求人工介入修单条坏数据；可接受。
- A3 修改 dict 字段命名可能影响 `format_markdown` / `format_json` / 飞书 sync 的其他读取处；必须 grep 一遍 `created_at` 在 export 链路中的所有读取点。
- A4 删除 `发现日期` 覆盖后，飞书已有记录里的 `发现日期` 值不会被回滚；只确保新行为正确即可，不做历史数据修复（如需修复另开任务包）。
