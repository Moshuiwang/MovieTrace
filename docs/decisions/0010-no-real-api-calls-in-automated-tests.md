# ADR-0010: 自动化测试不消耗真实外部 API

**状态:** Accepted  
**日期:** 2026-05-14  
**决策者:** 用户 + Claude Code (deepseek-v4-pro)  
**相关 Commit:** `fbf551e`  
**影响:** ADR-0009（API 预算与评分兜底策略的测试边界细化）

---

## 上下文

MovieTrace 的核心链路依赖 FlixPatrol、TMDb、Trakt、OMDb 等外部服务。到 Phase 1.10 后，外部 API 已经成为运行成本、配额和稳定性的主要约束之一：

- FlixPatrol 有明确月度调用预算，且 402/授权失败会造成重复无效请求。
- OMDb 免费 key 可能触发 401 / 配额问题，不能作为测试稳定前置。
- TMDb / Trakt 虽然可用性相对更好，但测试套件中的真实请求会让验证结果受网络、密钥、第三方状态影响。
- 2026-05-14 的测试优化中，直接调用 `run_discovery()` 的慢测试和两个集成测试被补齐 mock，测试耗时从约 257s 降到约 58s，且所有 API 消耗清零。

如果这个约束只留在日报里，后续 Agent 可能在新增集成测试时重新引入真实 API 调用，导致测试消耗配额、变慢或间歇失败。

## 决策

自动化测试默认不允许消耗真实外部 API。

- `pytest` 测试必须通过 mock / fixture / 本地样本数据覆盖外部 API 边界。
- 任何依赖 FlixPatrol、TMDb、Trakt、OMDb 等真实网络请求的验证，不属于默认自动化测试套件。
- 真实 API 验证只能通过明确的人工验收命令、dry-run、专项脚本或任务包授权执行，并且必须记录时间、命令、关键输出和外部失败风险。
- 新增测试如果需要覆盖 API 客户端行为，应 mock HTTP 响应或 secrets 读取，不得隐式读取真实 `.env` / secrets 并发起请求。

## 后果

**正面:**

- 默认测试结果更稳定，不受网络、密钥、配额和第三方服务状态影响。
- 保护 FP / OMDb / TMDb / Trakt 的调用预算，避免本地回归测试消耗真实额度。
- 测试耗时保持可控，便于高频运行。
- 外部 API 真实验收与单元/集成测试边界更清楚。

**负面 / 待解决:**

- 默认 `pytest` 不能证明当天真实 API 可用，只能证明本地逻辑和边界处理正确。
- API schema 漂移、授权变更、第三方错误码变化需要专项真实运行或 dry-run 才能发现。
- 后续如果引入 live smoke test，必须单独标记、默认跳过，并有任务包授权。

## 备选方案

### 备选 A：允许集成测试调用真实 API

- 优点：能更早发现真实 API schema 或授权变化。
- 缺点：消耗配额，测试受网络和第三方状态影响，失败原因不稳定。
- 拒绝原因：项目当前更需要可重复、低成本的默认验证；真实 API 可用性应通过专项验收覆盖。

### 备选 B：按环境变量启用 live tests

- 优点：保留一键真实 API 验证入口。
- 缺点：容易被 CI 或本地环境误触发，也会让测试语义变复杂。
- 拒绝原因：当前尚无明确 live smoke test 任务包；如未来需要，应另行设计并默认跳过。

### 备选 C：当前决策

- 优点：默认测试完全离线、稳定、低成本，符合当前 API 预算治理方向。
- 缺点：需要通过人工 dry-run 或专项脚本补足真实 API 健康检查。
