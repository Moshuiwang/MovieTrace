## 评分系统说明

### 当前评分分布（2026-05-11 数据）

所有 89 个候选的 hot_score 在 42-49 范围（全部 P3）。原因是：

- **外部信号缺失**：`api_cache` 表为空（0 rows），TMDb popularity / Trakt / TMDb rating / IMDb rating 四个因素（合计权重 45%）全部为 0
- **canonical_items 匹配率低**：仅 7/89 候选匹配到 canonical_items（Phase 0 导入 389 条，覆盖不足）
- **有效信号**：仅 FlixPatrol 排名（30%）、平台权重（10%）、内容类型（5%）、语言相关性（5%）四个因素有数据

这是 V1 MVP 的预期行为——评分系统透明展示哪些信号可用、哪些缺失。后续通过 P1-G CLI `refresh-signals` 补充外部数据后，hot_score 区分度会显著提升。

### candidates 表字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `canonical_item_id` | int/NULL | FK → canonical_items |

- 9 个独立因素函数，每个返回 0-100

## 剩余风险

1. **评分区分度不足** — 所有候选均为 P3，原因是外部信号缺失。这会在 P1-G `refresh-signals` 补充 TMDb/Trakt 数据后改善。P1-H 集成测试时可根据实际效果调整权重。

2. **canonical_items 覆盖率低** — 仅 7/89 候选匹配到 canonical_items。Phase 0 只导入了飞书基线 389 条，大量 FlixPatrol 热门内容不在基线内。P1-D 的基线匹配将使用 title+year 模糊匹配来扩大覆盖率。

3. **PyYAML 不可用** — 项目未安装 PyYAML，评分代码已优雅降级到默认硬编码权重。如需使用 YAML 配置文件调整权重，需先 `pip install pyyaml`（建议 P1-H 调优时安装）。

4. **`value` 字段未用于评分** — FlixPatrol 的 value 热度分（0-10）当前仅存储在 flixpatrol_top10 表中，未纳入 hot_score 公式（公式用 ranking + days_total）。value 可能提供额外区分度，但需要在 P1-H 调优时评估。
