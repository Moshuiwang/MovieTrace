## 公共接口（给 P1-C 使用）

### `load_api_key(secrets_path=...) -> str`

从 secrets JSON 文件加载 FlixPatrol API key。

### `FlixPatrolClient`

```python
client = FlixPatrolClient(api_key, timeout=60)

# 单 endpoint
items = client.fetch_top10(company, country, content_type, date_from, date_to)
# → list[dict]

# 全部 12 endpoint
results = client.fetch_all_platforms(date_from="2026-05-10")
# → dict[str, list[dict]]
# keys: "netflix/movie", "netflix/tv_show", "prime-video/movie", ...
```

### `unwrap_item(raw_item) -> dict`

从 JSON:API 复合文档格式提取扁平 dict。

### 返回 item 结构

```python
{
    "fp_id": str,           # FlixPatrol internal ID
    "title": str,           # 英文标题
    "content_type": str,    # "movie" | "tv_show"
    "ranking": int,         # 1-10
    "ranking_last": int,    # 昨日排名 (0 = 新上榜)
    "value": int,           # FlixPatrol 热度分
    "days_total": int,      # 累计在榜天数
    "platform": str,        # "netflix" | "prime-video" | ...
    "country": str,         # "united-states"
    "snapshot_date": str,   # "YYYY-MM-DD"
    "tmdb_id": int,         # TMDb ID (98.1% 覆盖)
    "imdb_id": int,         # IMDB ID
    "updated_at": str,      # ISO 时间戳
}
```

## 剩余风险

1. **Country name 来自硬编码映射** — API 的 country.data 不返回 name 字段，仅返回 id。当前通过 `COUNTRY_ID_TO_NAME` 常量映射 `cnt_iMUHNbZvnNHK5YdhgwtOoP4u` → `"united-states"`。如未来加更多地区需扩展。

2. **TMDb ID 缺口 4 条（1.9%）** — 这 4 条无 TMDb ID，P1-C 需要以 title+year 为 fallback 匹配到 canonical_items。

3. **端点返回量不一致** — Netflix/Prime Video/Disney+/Apple TV+/HBO Max 均返回 20 条 Movies，但 TV Shows 和 Hulu 返回量在 10-30 条之间。这是 API 正常行为（返回 date_from 之后所有匹配项），P1-C 评分时无需特别处理。

4. **`value` 字段语义** — FlixPatrol 热度分是平台内部值（0-10），当前直接存储。P1-C hot_score 评分公式中使用 value 作为排名因子的补充，权重由 P1-C 任务包决定。

5. **Apple TV+ 慢端点** — 实测 Apple TV+ 两个 endpoint 响应时间均 >20s，但在 60s timeout 内完成。Bootstrap 模式（180 天）需注意此延迟。
