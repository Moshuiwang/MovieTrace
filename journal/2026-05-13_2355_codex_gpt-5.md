## 关键决策记录

1. 降低 OMDb 依赖
   - 背景：OMDb 免费 key 当日触发 `Request limit reached!`。
   - 决策：OMDb 有结果时优先使用真实 IMDb 分；失败时不阻断评分链路。

2. 接受 TMDb fallback
   - 背景：IMDb ID 或 OMDb 评分缺失会压低 hot_score。
   - 决策：用 TMDb `vote_average` / `vote_count` 作为 IMDb 评分维度 fallback，并在 `score_breakdown` 标记 `tmdb_fallback`。

3. P1.8-F / P1.8-G 合并执行
   - 背景：两个任务都会实现 TMDb external_ids client、缓存和统计。
   - 决策：合并执行，报告分别覆盖 F/G 验收项。

4. P1.8 执行顺序
   - 决策：`P1.8-D → P1.8-H → P1.8-C → P1.8-F/G → P1.8-E`。
   - 理由：先有 API 用量日志，再扩 FP 和 external_ids，最后做大范围多源结构化。
